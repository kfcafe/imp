imp startup screen wireframe
============================

Goals
-----
- First paint should be fast and useful even while repo/workflow stats load.
- Recent sessions should live under the session panel, not consume a full column.
- Rules should render as white/default emphasis, not muted.
- Workflow panel should show real workflow state and recent activity, not /plan /run /verify command advertising.
- Tools should show native tools first, then Lua extensions underneath so extension capability is visible.
- Wide layouts should use the full available screen width instead of leaving a ragged right side.
- Avoid showing expensive counts unless cached or already loaded.

Wide layout notes
-----------------
- Use a real two-column grid across the full inner width.
- Top row: session + workflows, both wide enough for useful text.
- Bottom row: tools/extensions + skills/repo activity.
- Workflows get a wide column because their useful content is multi-field: status, next step, verify command, updated time.
- Recent sessions stay inside session because they are contextual to resuming work.
- Lua extensions are grouped under tools because they extend tool/command capability.

Progressive loading notes
-------------------------
- First paint:
  - project
  - session
  - provider
  - web
  - rules
  - actions
  - native tools from static registry
- Fast async:
  - primary language
  - LOC
  - file count
  - recent sessions if not already cached
  - Lua extension list if runtime metadata is cheap
- Slow/background or cached only:
  - symbols
  - tests
  - workflow aggregate counts
  - repo intelligence index
  - expensive extension metadata

Workflow panel data preference
------------------------------
1. Active workflow from .imp/workflows with status, next step, verify command, and updated time.
2. Recently updated workflows with status and age.
3. Summary counts only if cheap/cached.
4. Configured workflow profiles only if there is no durable workflow state.
5. Do not show /plan /run /verify as workflow content.

Open questions
--------------
- Should skills be hidden unless there are project-local skills?
- Should repo activity be shown, or should the lower-right panel be something else?
- Should Lua extensions show command names, extension names, or both?
- Should repo stats show counts at all, or only language + repo state?
