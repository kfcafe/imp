## Goal
Convert the architecture decisions from `.10.1` and `.10.2` into a tighter mana-first doctrine in imp prompt surface. imp already has some good mana guidance, but it still needs a clearer rule set for when durable plans, architecture, and synthesized project knowledge should be externalized into mana instead of staying trapped in session flow.

## Current State
- `config.rs` Orchestrator mode already says "externalize durable planning structure into mana during the conversation rather than waiting until the end."
- `system_prompt.rs` has a Mana doctrine block with 4 bullet points.
- Neither explicitly distinguishes the 4 memory layers or tells the agent what NOT to put in session history / personal memory.
- The existing `.9` epic expanded the mana tool surface but did not rework the doctrine wording.

## Scope
- In scope: prompt/doctrine changes in imp core and small supporting docs updates.
- In scope: clarify the distinction between session memory, personal memory, mana durable context, and any future wiki layer.
- In scope: keep this first pass prompt-level and testable.
- Out of scope: implementing a new runtime memory subsystem or UI feature.

## Deliverable
- A narrow code/docs patch that updates prompt guidance for Full / Orchestrator / Planner behavior.
- Regression coverage proving the new guidance is present where it should be and absent where it should not.

## Patch Guidance
- Prefer targeted edits in `config.rs`, `system_prompt.rs`, and adjacent docs.
- Do not bloat the system prompt with a giant design doc dump.
- Keep the doctrine actionable: when to externalize, what to externalize, and what not to externalize.

## Focus
- `crates/imp-core/src/config.rs` (mode instructions)
- `crates/imp-core/src/system_prompt.rs` (mana doctrine / layer wording / guidance blocks)
- `crates/imp-core/src/builder.rs` (only if wording or assembly needs adjustment)
- `README.md` / `ARCHITECTURE.md` for concise public explanation if needed

## Required Behavior
- Planning/design conversations should proactively externalize durable implementation structure into mana with minimal prompting.
- Guidance should explicitly say that personal memory files are not the project work graph.
- Guidance should leave room for a future wiki layer without pretending it already exists in runtime.
- Worker vs Orchestrator vs Planner boundaries should remain clear.

## Done / Verify
- Done when imp prompt/doctrine consistently pushes durable project structure into mana and cleanly separates it from session or personal memory.
