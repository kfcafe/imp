Goal: implement the first high-leverage imp-tui cockpit polish slice after the superbar/no-top-bar redesign direction is fully accepted.

Current state grounded this session:
- the TUI already has usable building blocks (top bar, status info, sidebar, editor, transcript rendering), but hierarchy and wording still feel like a generic terminal app.
- 248.16 captured the desired cockpit feel and the redesign has since narrowed to a superbar-centered layout.
- 248.16.1, 248.16.2, and 248.16.6 are expected to sharpen the target state model and refactor seams before implementation.

Implementation target for this slice:
1. Replace the standalone top bar with the chosen superbar treatment, moving identity and live status into the composer-centered frame.
2. Remove the generic widget tray/header and absorb mana/trust surfaces into the sidebar inspector.
3. Keep transcript tool rows compact while adding stronger scan/activity telemetry and animation where it improves operator awareness.
4. Recast the sidebar as a deep inspection rail for selected tool detail and mana/review/blocked states.
5. Tighten visible state language across the touched surfaces to the canonical vocabulary: idle, thinking, acting, streaming, queued, blocked, review, done, error.
6. Prefer '~' home abbreviation in visible filesystem path displays.

Ground against:
- imp/crates/imp-tui/src/app.rs
- imp/crates/imp-tui/src/views/{top_bar.rs,status.rs,editor.rs,sidebar.rs,chat.rs}
- imp/crates/imp-tui/src/animation.rs
- outputs from 248.16.1, 248.16.2, 248.16.6, and the accepted wireframe revision

Out of scope:
- detached second-TUI or terminal-multiplexer inspection experiments
- broad theme/palette redesign
- richer trust-state productization beyond the first integrated sidebar surfaces
