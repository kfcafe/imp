  Concrete divergence found (2026-04-22): imp's native `mana` tool is not equivalent to CLI `mana show` for archived units. In `imp/crates/imp-core/src/tools/mana.rs`, action=`show` calls `mana_core::api::get_unit(&mana_dir, id)`, which only resolves active units. By contrast the CLI `mana show` path goes through `mana_core::ops::show::get(...)`, which falls back from `find_unit_file` to `find_archived_unit`, so archived IDs like `266.4.9` resolve correctly there. Likewise native action=`tree` uses `mana_core::api::get_tree(...)`, which explicitly excludes archived units, so archived subtrees disappear even though CLI `show` can still inspect archived items. This explains the earlier cleanup confusion: the in-session native mana tool was stricter than the CLI, not because archive lookup was broken globally but because the native tool used the narrower API path.
