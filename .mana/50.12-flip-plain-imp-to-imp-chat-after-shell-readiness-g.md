## Task
Implement the staged default-entrypoint flip decided in `docs/rebuild/imp-default-entrypoint-migration.md`.

This unit is explicitly **not** the proving slice. It starts only after the proving slice and parity work have landed and shell readiness has been demonstrated.

## Decision contract
Plain `imp` must become the CLI-first interactive shell (`imp chat`), not the fullscreen TUI.

Keep `imp tui` as the explicit fullscreen path.

Do not introduce a fuzzy chooser that sometimes launches chat and sometimes fullscreen based on opaque heuristics beyond normal tty/stdin handling.

## Preconditions
Do not start this unit until these dependent implementation tracks are complete enough to support the default flip:
- `50.9` first CLI-first proving slice with explicit `chat` / `view` / `tui` entrypoints
- `50.10` guided CLI parity flows for settings, personality, and setup
- `50.11` first shell-to-view handoff for sessions, tree, logs, and checkpoints

## Shell-readiness gate
Before changing plain `imp`, verify that `imp chat` covers the minimum daily workflow already externalized in the migration decision:
1. streaming responses
2. start new session
3. resume session
4. model selection/control
5. thinking-level control
6. compact tool notices and summaries in the transcript
7. handoff from shell to `imp view`
8. explicit escape from shell to `imp tui`

Also confirm:
- transcript UX is credible without fullscreen dependence
- current TUI value remains reachable via `imp tui` or `imp view`
- docs/help have already moved to explicit command forms
- users have had at least one release window where `imp chat` / `imp tui` / `imp view` / `imp prompt` were visible before plain `imp` changes meaning

## Implementation steps
1. Update the no-subcommand interactive entrypoint in `imp/crates/imp-cli/src/main.rs` so interactive tty default behavior matches `imp chat`.
2. Preserve the standard piped-stdin no-subcommand behavior as the one-shot prompt/print path.
3. Keep explicit `imp tui` behavior intact and discoverable in help output.
4. Keep temporary compatibility aliases that the migration decision preserved (`imp -p` / `--print`, slash commands in shell, and any still-needed `imp web-login` alias).
5. Adjust top-level help and user-facing notices so the new default is clear and `imp tui` remains easy to find.
6. Record any temporary escape hatch only if it is intentionally short-lived and explicitly marked transitional.

## Acceptance
- interactive `imp` launches the CLI shell rather than fullscreen TUI
- `imp chat` remains the explicit equivalent of plain `imp`
- `imp tui` remains the explicit fullscreen path
- piped stdin without subcommand still behaves as a one-shot prompt path
- docs/help align with the flipped default rather than describing the old fullscreen default
- no hidden chooser logic is introduced

## Files
- `imp/crates/imp-cli/src/main.rs` (modify default-entrypoint behavior and help/user notices)
- related CLI help/tests under `imp/crates/imp-cli/` (modify/add as needed)
- docs under `docs/rebuild/` and user-facing `imp/README.md` if implementation-phase alignment is needed

## Do not
- do not flip the default early just because explicit subcommands exist
- do not remove `imp tui`
- do not silently strand settings/personality/setup or browse-heavy flows
- do not introduce heuristic mode switching between chat and fullscreen
