This epic formalizes the shared boundary between mana and imp as code. Goal: replace the current partly implicit boundary with a canonical protocol layer that both sides can depend on.

Focus:
- assignment types
- task/plan/evidence/result types
- durable vs ephemeral ownership rules
- compatibility adapters during migration

Important decisions to resolve explicitly:
1. should shared contracts live at Tower root, under mana, or under imp?
2. which current structs in `imp-core/src/mana_worker.rs` and nearby areas should migrate first?
3. what must become durable substrate state vs what remains runtime-local?
4. how strict should typed artifact/provenance requirements be in the first pass?

Suggested child jobs:
- define first canonical shared types (`TaskSpec`, `WorkerAssignment`, `WorkerOutcome`, `VerifierResult`, `EvidenceBundle`)
- map existing imp runtime types to the new contract shapes
- identify all currently misplaced durable objects that still live only in imp
- add compatibility adapters/re-exports so migration can happen incrementally

Done when:
- there is a concrete contract shape and migration map that prevents further boundary drift
- the first implementation slice can start without guessing where the protocol belongs
