Goal: design how imp and/or Wizard should expose mana state inline so the user can review the persistent work/idea graph between turns.

Current state: even when mana is used, the user does not automatically get good visual review of the evolving unit graph during ordinary conversation turns.

Steps:
1. Define the minimum mana information that should be visible between turns.
2. Define whether this belongs in imp TUI, Wizard, or both.
3. Define the interaction loop for reviewing, editing, approving, or pruning mana structure during active conversations.
4. Identify what can ship as a simple first step vs later richer canvas-like behavior.

Files:
- wizard/ (relevant orchestration/view crates)
- imp/crates/imp-tui/
- imp/crates/imp-core/src/tools/mana.rs

In scope:
- UX/system behavior proposal
- review loop design

Out of scope:
- visual polish implementation

Do not:
- assume all users want intrusive UI
- require full Wizard dependence for the first version
