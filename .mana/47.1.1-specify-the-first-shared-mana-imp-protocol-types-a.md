  Repo-state findings after inspection:

  1. The crate-placement decision is already substantially answered by code and docs.
  - `tower-contracts/` already exists at the Tower root.
  - `imp/crates/imp-core/src/mana_worker.rs` already re-exports `tower_contracts::worker::{WorkerAssignment, WorkerAttempt, WorkerResult, WorkerStatus}`.
  - `docs/rebuild/tower-contracts-surface.md` and `docs/rebuild/imp-worker-boundary-plan.md` already frame `tower-contracts` as the neutral shared boundary.
  Therefore the first implementation slice should treat crate placement as decided unless new contrary constraints appear.

  2. The first stable v1 type set should stay narrow.
  Already present in `tower-contracts` and suitable as first-class v1 shared contracts now:
  - WorkerAssignment
  - WorkerAttempt
  - WorkerResult
  - WorkerStatus

  3. The next types to add after the worker quartet should be introduced only when a second consumer exists or the migration order requires them.
  Recommended follow-on order:
  - VerifierResult (because mana_worker already runs verify/close flows and evidence/completion semantics need a shared result shape)
  - EvidenceBundle or EvidenceRef vocabulary (minimal reference-level shape, not full storage semantics)
  - runner request/result vocabulary once 47.3 is ready to consume it
  TaskSpec can stay deferred unless we prove that WorkerAssignment is too execution-shaped to serve as the first durable task packet.

  4. Current first mapping target remains `imp-core/src/mana_worker.rs`.
  That file is both the current consumer and the current adapter seam. It already anchors:
  - assignment loading from mana-core
  - task/context shaping
  - verify/close result handling
  - shared worker result production
  So 47.1.2 should start by formalizing adapter cleanup around this file rather than searching for a new center.

  5. Compatibility rule for the next slice.
  Acceptable temporarily:
  - keep re-exports from `imp_core::mana_worker` so existing call sites compile
  Target state:
  - call sites that only need worker contracts import directly from `tower_contracts::worker`
  - `mana_worker.rs` keeps execution logic and translation helpers only

  6. Important scope guard.
  Do not move broad mana domain semantics, scheduler internals, full artifact storage, or UI projection types into `tower-contracts` in the first slice. The first slice should remain a narrow worker/protocol seam, not a wholesale shared-domain migration.
