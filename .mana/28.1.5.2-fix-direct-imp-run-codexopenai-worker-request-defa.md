Goal: make direct headless `imp run` work with the OpenAI Codex-style path by ensuring the request includes the provider-required instruction/system content.

Current state:
- A smoke test on 2026-04-15 confirmed `../target/debug/imp --provider openai-codex -m gpt-5.3-codex ... run 49 --defer-verify` reaches worker startup and the provider call.
- The provider then fails with HTTP 400 `Instructions are required`.
- This indicates the direct worker path is reaching the provider, but the request construction/default prompt mapping for the Codex/OpenAI path is missing required instruction/system content.

Steps:

4. Add focused tests covering the request shape or serialization contract that failed here.
