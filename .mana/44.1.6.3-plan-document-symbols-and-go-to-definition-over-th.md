## Goal
Plan the symbol-navigation slice for Phase 1 read-oriented code intelligence in `imp`: document symbols and go-to-definition.

## Current state
The read-oriented plan places document symbols and go-to-definition after diagnostics and after the common normalization seam. These should provide reliable symbol navigation without requiring write-oriented operations.

## Scope
- Planning only.
- Cover document-symbol results, definition targets, location/symbol rendering, and compact transcript behavior.
- Preserve AST extraction as a context companion rather than replacing semantic navigation.

## Acceptance
- The slice identifies the expected model-facing outputs for document symbols and definition targets.
- It records how semantic navigation should compose with AST extraction for nearby context and enclosing blocks.
- It defines when results should stay inline versus hand off to a viewer/inspector path.

## Important constraints
- Do not broaden this slice to workspace-wide result browsing or references explosion.
- Do not assume one semantic backend protocol is permanently chosen.
- Keep freshness/provenance visible on returned results.
