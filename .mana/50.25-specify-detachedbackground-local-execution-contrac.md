Write a rebuild-spec doc for detached/background local execution in CLI-first imp, explicitly sequenced after stable output modes, browse/search surfaces, and approval policy. Ground it in the current runner/runtime proposals and the CLI-affordance sequence note. The doc should define the minimum contract for `imp run --detach` or equivalent: run identity, list/status/reattach/log inspection, structured intermediate/final states, approval-required / awaiting-input / waiting states, and how detached local work relates to mana-backed unit execution versus plain session-backed work. Acceptance criteria:
- states prerequisites from the CLI-affordance sequence explicitly
- defines list/status/reattach/log concepts for detached local work
- includes approval/input waiting states, not only success/failure
- links detached behavior to stable machine output and inspectable session/view surfaces
