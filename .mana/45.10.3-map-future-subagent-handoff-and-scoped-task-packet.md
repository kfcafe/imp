---

Goal: preserve and improve the current high-value mana workflow where imp points a subagent at a bounded job and the worker receives exactly the context it needs without making decisions outside scope.
Current state: the future architecture is shifting toward `mana` as platform, `imp` as agent, and a broader work-memory graph rather than only file-native unit/fact records. The refactor must not lose the strong existing handoff pattern where a subagent executes a bounded mana job with a purpose-built context window and clear scope.

Explore and document:
1. How subagents should attach to a task in the future graph model.
2. What the scoped task packet should contain (task contract, graph slice, evidence, approvals, allowed actions, verify/close contract, constraints, relevant artifacts).
3. Which decisions belong to the parent/planner versus the subagent worker.
4. How packet assembly should use the graph so the worker inherits exactly what it needs and not unrelated project state.
5. How to preserve compatibility with current mana-job worker flow during migration.
6. What this implies for the work-memory graph design and retrieval surfaces.

In scope:
- architecture and packet design
- subagent scope boundaries
- future migration alignment

Out of scope:
- implementation
- changing current code today

Do not:
- let the subagent become a freeform planner over the whole project
- collapse task scoping into raw transcript replay
- lose verify/approval/evidence constraints during handoff
