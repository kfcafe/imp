Goal: keep the generated prompt preview available on the startup panel without letting it dominate the launch experience.

Current state:
- `crates/imp-tui/src/views/startup.rs` renders a bounded generated prompt preview when enough space exists.
- The preview is useful for transparency, but it competes with launch guidance and actionability on first view.
- Prior product direction already established that this preview should only include internally generated prompt layers, not file-backed context like AGENTS.md or discovered prompt files.

Steps:
1. Rework the startup panel so the prompt preview is clearly secondary: collapsed by default, toggleable, or otherwise visually demoted while still accessible.
2. Preserve the existing product constraint that only internally generated prompt layers are previewed.
3. Ensure narrow layouts still prioritize actionable startup content over prompt internals.
4. Add focused tests for any new preview-state shaping logic if introduced.

Files:
- `crates/imp-tui/src/views/startup.rs` (modify — adjust preview presentation and any stateful affordance text)
- `crates/imp-tui/src/app.rs` (modify if panel data/state needs to support preview visibility)

In scope:
- visual priority and accessibility of the prompt preview
- bounded preview behavior

Out of scope:
- changing how the actual agent system prompt is assembled
- exposing file-backed prompt layers in the launch screen

Do not:
- remove transparency entirely
- make the preview unbounded or primary by default
