Goal: Add Z.AI API-key provider support to imp using Pi model/provider compatibility and official Z.AI docs.
Current state: Known evidence from official docs says Z.AI uses API key or JWT-token auth, not OAuth. Base endpoint: `https://api.z.ai/api/paas/v4`; coding endpoint: `https://api.z.ai/api/coding/paas/v4`; auth header: `Authorization: Bearer YOUR_API_KEY`; env var in Pi: `ZAI_API_KEY`. imp currently does not have first-class Z.AI provider entries. This unit should implement API-key support only unless the plan doc contains new official OAuth evidence.
