## Task
Wire the canonical between-turn mana review packet into imp-cli and RPC-friendly output so the same semantics survive outside the TUI.

## Goal
Make CLI/RPC consumers show or emit the same between-turn mana review semantics as the TUI without inventing a separate contract.

## Grounded context
- `docs/rebuild/imp-between-turn-mana-review-ui.md` says the textual inspector should stay presentation-neutral enough that CLI/RPC can mirror it.
- `imp/crates/imp-cli/src/main.rs` already forwards status/widget traffic over its RPC/event path.
- `docs/rebuild/mana-between-turn-review-affordance.md` and the UI spec both require explicit `no_change`, `changed`, and `needs_decision` states and question-gating only on consequential unresolved choices.

## Required behavior
1. Render or emit the compact review block semantics for CLI/RPC consumers from canonical TurnManaDelta input.
2. Preserve explicit `no_change`, `changed`, and `needs_decision` states.
3. Provide an expanded line-oriented textual inspector form that mirrors the TUI semantics when requested or surfaced.
4. Preserve question-gating so only consequential unresolved choices trigger a concise question.
5. Avoid falling back to raw per-action mana tool output as the primary review signal.

## Files
- `docs/rebuild/imp-between-turn-mana-review-ui.md` (read)
- `imp/crates/imp-cli/src/main.rs` (modify)
- shared imp-core TurnManaDelta formatter/config from dependencies (read/use)
- CLI/RPC-facing status/widget/event plumbing already in imp-cli (read/use)

## Acceptance criteria
- CLI/RPC consumers can show compact between-turn mana review semantics from canonical TurnManaDelta input.
- Line-oriented expanded review text is available without TUI-specific assumptions.
- Consequential unresolved choices are the only review items that force a user-facing question.

## Do not
- Do not create a separate CLI-only mana review schema.
- Do not require fullscreen UI.
- Do not let raw per-action mana widget strings remain the only visible review path outside the TUI.
