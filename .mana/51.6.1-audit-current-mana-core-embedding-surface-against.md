## Task
Write `docs/rebuild/mana-embedding-surface-audit.md` auditing the current `mana-core` embedding surface against the target semantic, lease-based, library-first contract captured in root mana.

## Current inspected state
- `mana/crates/mana-core/src/api/mod.rs` already exposes broad query/mutation helpers and re-exports orchestration/query types.
- `mana/crates/mana-core/src/ops/mod.rs` shows several embedding-relevant operations still living only under lower-level ops modules.
- `mana/crates/mana-core/src/ops/run.rs` currently exposes ready-queue / run-plan scheduling helpers rather than a run/lease/event/executor contract.
- `mana/crates/mana-cli/src/commands/run/mod.rs` still documents `mana run` as a CLI entrypoint with template/direct spawn behavior.
- `docs/rebuild/mana-lease-model.md` already defines the first durable lease vocabulary.
- Root mana notes on `51.6` now capture the target contract shape: semantic intents rather than raw state mutation; minimum durable concepts Run, Lease, Event, ArtifactRef / CheckpointRef, Capabilities; useful semantic ops including `create_run`, `get_snapshot`, `stream_events`, `attach_executor`, `heartbeat`, `append_evidence`, `resolve_lease`, and `control_run`.

## Steps
1. Inspect `mana/crates/mana-core/src/api/mod.rs`, `mana/crates/mana-core/src/ops/mod.rs`, `mana/crates/mana-core/src/ops/run.rs`, and `mana/crates/mana-cli/src/commands/run/mod.rs`.
2. Compare current repo reality to the target semantic contract recorded on `51.6`.
3. Identify which current surfaces already align, which are missing entirely, and which are the wrong layer (CLI-shaped, template-shaped, or status-CRUD-shaped).
4. Recommend the smallest coherent next embedding slice:
   - either add a narrow set of top-level `mana_core::api::*` wrappers / service types for the embedding-critical contract pieces now, or
   - explicitly defer wrappers and define one stable full-spawn/library boundary first.
5. Name the minimum proving slice and verify direction for the recommended next step.

## Acceptance
- The audit is grounded in the inspected files above.
- It evaluates the concrete target concepts and semantic operations already recorded on `51.6` rather than inventing a new contract.
- It explicitly answers which pieces should become embedding-critical next, which can remain lower-level/CLI compatibility for now, and where the first stable library-facing spawn/attach boundary should live.
- It includes a narrow recommended next proving slice and a future verify direction.

- Do not invent a cloud/service architecture.
