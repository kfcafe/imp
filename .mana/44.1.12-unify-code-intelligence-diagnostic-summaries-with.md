## Goal
Integrate Phase 1 read-oriented code-intelligence outputs into the operator experience without flooding the transcript with raw protocol noise.

## Current state
- The plan doc requires a unified diagnostic view that can aggregate semantic diagnostics and explicit build/test/linter output while preserving per-item provenance.
- `imp` already has CLI-first transcript work and TUI surfaces that favor compact notices over giant raw dumps.

## Scope
- unified diagnostic summary behavior
- provenance-aware merging of semantic diagnostics with check output
- compact CLI/TUI transcript summaries and inspect-on-demand behavior
- preserving structured observations for follow-up turns

## Acceptance
- The runtime can answer 'what is broken here?' from one summary surface without losing the distinction between semantic diagnostics and explicit checks.
- Default transcript output is compact and operator-legible.
- Raw payloads, if retained, stay behind inspect/debug layers rather than becoming the primary session surface.

## Depends on
- the shared read-oriented query/result contract
- initial diagnostic surfaces
