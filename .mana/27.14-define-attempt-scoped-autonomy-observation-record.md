Goal: define and implement the attempt-scoped observation record that lets imp contribute visible-evidence inputs to future scheduler gating without exporting raw confidence.

Current state:
- `imp/crates/imp-core/src/agent.rs` has an imp-local continue hook that only fires after visible mana use and positive completion/next-step language.
- `ContinuePolicy` in `imp/crates/imp-core/src/config.rs` is operator-facing runtime policy.
- 27.10 concluded that imp should keep confidence computation locally and only publish typed visible-evidence observations upstream.

Task:
Introduce the attempt/result-side observation shape that imp can populate for mana-core policy evaluation.

Concrete steps:
1. Inspect the current attempt/result structures in `mana/crates/mana-core/src/unit/types.rs` and the imp runtime continue hook in `imp/crates/imp-core/src/agent.rs`.
2. Define a typed observation record for attempt-scoped autonomy evidence, carrying fields such as:
   - visibility state (`satisfied`, `missing`, `not_applicable`, `unknown`)
   - provenance (`attempt_observation`, `close_evidence`, `mixed`, `unknown` or equivalent)
   - source attempt number / timestamp
   - optional continuation budget contribution if the contract keeps one
3. Ensure the shape explicitly excludes raw confidence, model self-report, and free-text heuristic export.
4. Add/update tests or serialization coverage as needed.
5. Leave actual policy evaluation to mana-core and leave pool consumption for later work.

Files:
- `mana/crates/mana-core/src/unit/types.rs` (modify)
- any adjacent integration point needed for attempt/result durability in `mana-core` (inspect first)
- `imp/crates/imp-core/src/agent.rs` (read for context; only modify if required to wire the typed observation shape, not to change user-facing continuation behavior)

Do not:
- Do not export raw confidence or confidence thresholds.
- Do not let mana-pool read private imp heuristics.
- Do not replace current imp-local continue behavior.

Context from 27.10:
- imp owns evidence gathering and local confidence heuristics.
- mana-core owns normalization into the scheduler-facing disposition.
- mana-pool should only consume the derived explicit disposition later.
