Goal: define and implement the migration/reconcile behavior from current XDG/macOS/legacy imp roots into canonical `~/.imp/`, after the shared storage-path helpers exist.

Current state:
- existing installs may have durable files spread across `~/.config/imp`, `~/.local/share/imp`, `~/Library/Application Support/imp`, and legacy `~/.imp`
- canonical accepted destination is `~/.imp/`

Steps:
1. For each major surface (config/auth metadata, sessions, indexes, soul/memory/user, skills/prompts/tools/lua), define source candidates and canonical destination.
2. Implement a non-destructive reconcile/migration layer that can discover old roots and populate canonical `~/.imp/` safely.
3. Define clear precedence when the same logical artifact exists in multiple old roots.
4. Add focused tests for migration/reconcile behavior.
5. Do not remove old-path compatibility until the migration behavior is proven.

Files:
- shared storage-path module from 264.3.1
- `imp/crates/imp-core/src/config.rs` or a nearby migration module
- relevant callers/tests

In scope:
- non-destructive migration/reconcile behavior
- conflict handling for legacy roots

Out of scope:
- documentation polish until behavior is settled
