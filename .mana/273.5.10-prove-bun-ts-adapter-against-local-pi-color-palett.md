Goal: prove the Bun-backed TypeScript extension adapter against a real local Pi extension instead of only synthetic fixtures.

Candidate:
- `~/.pi/agent/extensions/color-palette` is the first local smoke target. It is self-contained and exercises real Pi extension patterns, but it is redundant with imp's native `color_palette` tool, so treat it as a mechanics/compatibility fixture rather than a product-value target.

Steps:
1. Import or fixture-copy `color-palette` into `.imp/extensions/pi/color-palette` in a temp/project test environment.
2. Load it through the Bun-backed TypeScript extension adapter.
3. Verify that its tool registers with the expected name and schema.
4. Invoke a simple action if feasible and compare output shape to expected imp `ToolOutput`.
5. Record any unsupported API/import/schema failures as follow-up compatibility tasks.

Out of scope:
- Claiming color-palette should replace imp's native tool.
- Full local Pi extension suite compatibility.

Acceptance:
- A test or documented smoke command proves whether `color-palette` can register and execute through imp's TS adapter, with failures captured as concrete compatibility gaps.
