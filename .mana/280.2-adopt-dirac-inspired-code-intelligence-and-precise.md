Goal: Turn the Dirac comparison into executable imp tooling specs that workers can implement cold. Dirac's strongest inspected advantages are AST/symbol-native code context, hash/anchor-backed edits, same-turn edit batching, diagnostics feedback, and benchmark discipline. imp should adopt those primitives on top of its existing mana/runtime/checkpoint strengths, not clone Dirac's IDE-first architecture.

Current evidence:
- Dirac exposes `get_function`, `get_file_skeleton`, `find_symbol_references`, `replace_symbol`, `rename_symbol`, `edit_file` with anchor protocol, and batch edit processing.
- Dirac prompts prioritize surgical code inspection over full-file reads and require batching non-overlapping edits.
- imp has `scan`, `read`, `edit`, `multi_edit`, checkpoints, mana, and worker evidence. Local inspection confirmed:
  - `crates/imp-core/src/tools/scan/mod.rs` already has `extract_symbol`, `extract_blocks_at_lines`, `ScanTool` with `scan/build/extract` actions.
  - `crates/imp-core/src/tools/edit.rs` is exact/fuzzy text replacement and delegates `edits` arrays to `multi_edit`.
  - `ToolContext` already carries `file_cache`, `checkpoint_state`, and `file_tracker` for read/stale/checkpoint behavior.
- Missing: first-class symbol tools, anchor-backed read/edit protocol, edit transaction/batching semantics, diagnostics loop, and eval harness/reporting.

Architecture choice:
- Build native imp/runtime capabilities with narrow tool surfaces.
- Reuse existing scan/tree-sitter and checkpoint/file-tracker infrastructure where possible.
- Preserve exact-text `edit` for simple cases; add anchor/symbol tools alongside it rather than breaking current behavior.
- Keep VS Code/IDE integration out of the first slice.

Success criteria:
- Child specs define concrete, scoped implementation slices with file paths, boundaries, embedded context, and verify gates.
- Specs are usable by cold worker agents.
- Specs preserve imp's architecture direction: runtime/mana/checkpoint/evidence first; code-intelligence tools as native/runtime capabilities.
