Goal: make resume/re-entry a first-class path from the normal startup panel by showing recent sessions directly on launch.

Current state:
- The empty-state startup panel shows generic session metadata, but not a small list of recent resumable sessions.
- The TUI already has a richer session browser in `crates/imp-tui/src/views/session_picker.rs` and session operations in `crates/imp-tui/src/app.rs`.

Steps:
1. Reuse existing session summary information to surface the top 1-3 recent sessions directly in the startup panel.
2. Show enough information to help a user choose: title/summary, project hint, and recency.
3. Keep the panel lightweight; this is a launch affordance, not a full session browser.
4. Make the startup copy clearly indicate that full browse/search still lives in the session picker.

Files:
- `crates/imp-tui/src/app.rs` (modify — gather recent session summaries for startup rendering)
- `crates/imp-tui/src/views/startup.rs` (modify — render recent-session previews compactly)
- `crates/imp-tui/src/views/session_picker.rs` (read — follow existing summary/ranking patterns)

In scope:
- showing recent resumable sessions on launch
- compact preview formatting

Out of scope:
- redesigning the full session picker
- changing session persistence or ranking rules broadly

Do not:
- duplicate the full resume browser inside the startup panel
- expand this into new session-management features
