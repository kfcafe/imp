Goal: make the first evidence model concrete enough to implement. Current state: the rebuild strategy calls for verification/review/evidence to become first-class, but the minimum bundle and status model are not yet explicit.

In scope:
- define the minimum evidence bundle for a non-trivial coding run
- specify completion statuses and what each requires
- distinguish ephemeral runtime evidence from durable mana records
- define what the skeptic/review stage must consume and emit first

Out of scope:
- implementing the entire verify/review pipeline
- designing a complete future review UI

Important decisions to resolve explicitly:
1. what is mandatory evidence vs optional evidence in v1?
2. how should `verified_complete`, `complete_with_gaps`, `needs_review`, and `failed_verification` map into mana state?
3. what review outputs need to be durable immediately?
4. what artifacts can remain implementation detail of the runner/runtime?

Suggested follow-ups:
- add shared evidence/artifact types
- separate edit output from verify output in the worker path
- add skeptic/review artifact schema
- connect mana-side durable state to the chosen completion statuses
