Implement the first explicit shell-to-view browsing handoff after the viewer submodes and proving slice are settled. Ground this work in 50.7.1, 50.7.1.1, 50.7.1.2, plus the current browse-oriented components (`session_picker`, `tree`, `sidebar`). Scope:
1. introduce focused `imp view` entrypoints for the first browse-heavy tasks, likely `sessions`, `tree`, `logs`, and `checkpoints`
2. wire shell handoff commands such as `:view sessions`, `:view tree`, `:view logs`, and `:view checkpoints` to those entrypoints with clear process/mode boundaries
3. keep the shell transcript-oriented and avoid silently mutating it into a browser
4. preserve return/back semantics clearly enough that the user understands when they are in the shell versus the viewer
5. reuse existing browse-oriented code where practical, but let the product surface be explicit and CLI-legible

Out of scope:
- making `imp view` the default authoring path
- full unification of every current TUI surface into the viewer before proving the first focused modes
- deleting the optional TUI altogether
