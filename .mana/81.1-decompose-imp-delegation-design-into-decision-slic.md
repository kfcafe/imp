## Task
Break the imp-native delegation design into the core decisions another worker would need in order to implement the first slice without inventing architecture.

## Context
Parent 81 is designing the first imp-native delegation/tool surface around the existing canonical `imp run <unit-id>` worker path.

Current likely direction:
- `imp` owns live child-worker invocation/runtime.
- `mana` owns durable orchestration state and inherited outcomes.
- v1 likely uses a two-layer model: transient ad hoc delegation may be possible, but any durable or mutating delegation must route through mana and reuse `imp run`.

## Required decomposition
Create child units under 81 that separately answer:
1. canonical delegation target model (`mana` unit vs ad hoc prompt vs two-layer)
2. parent-facing result contract (status/events/log refs/artifacts)
3. approval, isolation, and policy constraints for nested imp invocation
4. migration path tying the new tool to `mana run` compatibility and longer-term runner protocol

## Acceptance
- Child units are worker-ready and mutually non-overlapping.
- Descriptions cite the currently inspected files and the parent design direction.
- Sequence/dependencies make architectural order obvious.
