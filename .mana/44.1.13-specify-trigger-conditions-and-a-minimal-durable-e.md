## Goal
Decide whether Tower should introduce a small typed shared envelope for **promoted** code-intelligence evidence, and if so specify the narrow handoff shape without pulling live runtime orchestration into `mana`.

## Current state
- `docs/design/code-intelligence-evidence-promotion.md` establishes the new durable policy: code-intelligence results are transient by default and only promoted consequences should survive into `mana` as notes, verify evidence, facts, or durable decisions.
- That note explicitly says a shared live code-intelligence runtime/schema is **not** warranted now.
- It also recommends that a small typed contract may become worthwhile later only if promoted summaries become frequent/inconsistent enough or if review/approval/promotion flows need stronger structured fields.

## Scope
- planning/spec only
- define the trigger conditions for introducing a shared durable envelope later
- evaluate where such a handoff type should live if it becomes necessary
- specify only the minimal promoted-evidence fields needed for durable review/approval/handoff

## Required decisions
- What concrete implementation pressure should trigger typed promotion envelopes later
- Whether any future handoff type belongs in `mana-core`, `tower-contracts`, or an `imp`-local projection layer over durable data
- What minimal fields are worth typing (for example source class, tool/backend, scope, status, severity counts, primary blocker, representative locations, artifact ref, freshness/trust posture)
- What must remain out of scope (raw payloads, live session state, backend lifecycle, query orchestration)

## Deliverable
- `docs/design/code-intelligence-promoted-evidence-envelope.md`

## Do not
- Do not propose a shared live code-intelligence subsystem across `mana` and `imp`
- Do not recommend storing raw LSP payloads or per-turn diagnostic streams in `mana`
- Do not move runtime orchestration into `mana`

## Acceptance
- The doc gives explicit trigger conditions for when Tower should keep using textual summaries versus when to introduce a minimal typed durable envelope.
- The recommended envelope, if any, is limited to promoted handoff objects rather than live runtime state.
- Ownership and placement options are compared against the current mana ↔ imp boundary.
