## Current State
imp already has UI abstractions for `set_status`, `set_widget`, and compact runtime surfaces, and the TUI work in unit `30` is intended to render those surfaces cleanly. This unit should build the mana-specific UX on top of that plumbing so background/native mana work is legible without flooding the transcript.

## Task
Add a mana-specific compact status and progress surface in imp.

Implement a small mana UI surface that:
1. shows active mana work in a compact area
2. updates during start, progress, success, and failure transitions
3. keeps the transcript concise while mana work runs in the background
4. reuses the existing UI abstraction rather than inventing new UI primitives
