Goal: implement the imp-side reusable unit worker runtime and stronger `imp run` contract after the architecture spec lands.

Current state:
- `run_headless_mode()` in `imp/crates/imp-cli/src/main.rs` is a large CLI-local flow that mixes unit loading, context assembly, agent startup, output formatting, verify behavior, and mana shell handoff.
- We want `imp run` to become the canonical single-unit worker runtime that `mana run` dispatches in parallel.
- This should likely involve extracting a reusable imp-core component rather than keeping worker logic embedded in imp-cli main.

Steps:
1. Read the architecture spec from 28.1.1 and implement the imp-side worker/runtime contract it defines.
2. Extract a reusable worker/unit-runner component into imp-core or another appropriate shared crate/module.
3. Update imp-cli `run` to wrap that runtime instead of owning all worker logic directly.
4. Add the machine-readable worker mode/protocol and human-facing mode behavior required by the spec.
5. Add focused tests around worker outcomes, protocol output, and verify/defer behavior.

- strengthened `imp run` CLI contract
- worker-json or equivalent machine-readable protocol if the spec calls for it
- focused tests

Out of scope:
- `mana run` integration changes outside the contract it consumes
- broader imp TUI polish beyond what the worker contract requires

Do not:
- keep long-term worker logic trapped only in imp-cli main
- leave orchestrated worker completion dependent on shelling out to `mana close` if the spec replaces that with structured outcomes
