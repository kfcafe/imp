Goal: Make imp API-key auth robust when auth metadata exists but the OS keychain value is missing, and make the CLI clearly diagnose/repair that state.

Current state:
- `imp secrets show google` reports metadata: `google` has field `api_key` in `~/.imp/auth.json`.
- Runtime auth fails with `No secret field 'api_key' found for google`.
- Direct macOS keychain probe confirms the value is absent: `security find-generic-password -s imp -a google:api_key` returns `SecKeychainSearchCopyNext: The specified item could not be found in the keychain.`
- This blocks Gemini-backed Dirac eval execution in unit 280.1.

Steps:
1. Inspect `crates/imp-llm/src/auth.rs` SecretFields resolution and `crates/imp-cli/src/lib.rs` `secrets show/set/remove` flows.
2. Change `imp secrets show <provider>` so it distinguishes metadata-only from value-present without printing values. Example: field status `api_key: missing` or a warning that secure storage value is missing.
3. Add a repair path or clear instruction. Prefer minimal CLI UX: `imp secrets set google` should rewrite both metadata and keychain value; if metadata exists but keychain is missing, successful set must leave `resolve_secret_field("google", "api_key")` working.
4. Improve missing-secret error text to mention stale metadata/keychain mismatch when metadata has the field but backend returns no value.
5. Add tests using the in-memory/fake auth backend already present in auth tests; do not require macOS keychain in tests.

Files:
- `crates/imp-llm/src/auth.rs` (modify — diagnostics and tests around SecretFields with missing backend values)
- `crates/imp-cli/src/lib.rs` (modify — secrets show/set UX if needed)

In scope:
- API-key SecretFields metadata/value mismatch diagnostics.
- Google is the observed provider, but fix should be provider-generic.

Out of scope:
- Adding Gemini Flash 3 Preview model metadata.
- Running Dirac evals.
- OAuth provider work.

Do not:
- Print secret values.
- Delete user metadata automatically without explicit remove/repair action.
- Depend on macOS Keychain in tests.
