Goal: replace ad hoc headless worker selection/auth branches with a shared runtime connection resolver that is model-first and route-aware.

Current state:
- `imp-cli` and `imp-core::imp_session` each contain model/provider/auth selection logic.
- OpenAI routing currently leaks `openai` vs `openai-codex` pseudo-provider branching into top-level runtime code.
- The desired architecture is model-first: the caller expresses desired model/thinking/defaults, then a shared resolver chooses the internal runtime surface/route and credential strategy.

Steps:
1. Inspect current selection/auth code in `crates/imp-cli/src/lib.rs` and `crates/imp-core/src/imp_session.rs`.
2. Design and implement a shared runtime connection abstraction with inputs for desired model, thinking, explicit route/provider override, normalized API key override, available auth, and default inheritance context.
3. Return a resolved connection bundle that includes model id, chosen runtime surface/route, and credential requirement/result.
4. Refactor headless worker/session startup to consume that shared resolver instead of duplicating branches.
5. Keep transport-specific provider implementations separate below this layer.
6. Add focused tests for route selection and default inheritance behavior.

Files:
- `crates/imp-cli/src/lib.rs`
- `crates/imp-core/src/imp_session.rs`
- `crates/imp-core/src/mana_worker.rs`
- `crates/imp-llm/src/model.rs`

In scope:
- shared runtime connection selection
- default inheritance logic for headless worker startup
- removing duplicated top-level routing branches

Out of scope:
- transport-specific request-shape fixes except where needed to fit the new resolver
