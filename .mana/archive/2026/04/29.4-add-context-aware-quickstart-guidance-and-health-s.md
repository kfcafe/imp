Goal: make the startup panel explain both what imp already knows here and how to begin usefully in this repo/session.

Current state:
- The startup panel already shows tools/skills/extensions/session/runtime context, but it reads mostly as an access-surface inventory.
- The app can already detect substantial local context such as cwd, model/provider state, discovered skills, and session state.

Steps:
1. Add a compact detected-context/health section that highlights the most decision-relevant state: project/repo label, configured provider/auth state, current model/thinking, and notable local context availability where evidence is already assembled.
2. Add starter guidance or prompt examples that help a user begin real work from this repo, preferably phrased as outcomes rather than tool names.
3. Favor context-aware examples when there is clear local evidence; otherwise fall back to concise generic examples.
4. Keep the panel readable and scannable; do not turn it into a full diagnostics dump.

Files:
- `crates/imp-tui/src/app.rs` (modify — assemble context health and quickstart example data)
- `crates/imp-tui/src/views/startup.rs` (modify — render health summary and starter guidance)
- `crates/imp-tui/src/views/welcome.rs` (read — align copy tone where helpful)

In scope:
- detected context summary
- startup examples/guidance tailored to current state when practical

Out of scope:
- new detection backends
- broad configuration or auth redesign

Do not:
- expose file-backed prompt context that the startup panel intentionally excludes
- overload the screen with low-signal debug details
