Goal: expose the new checkpoint capability through a minimal user-facing TUI command surface after the core and session helpers exist.
Current state: the TUI already has a slash-command palette and App::execute_command() handling for commands like /compact, /fork, /memory. There is no checkpoint discoverability or restore/list surface yet.

Steps:
1. Add one or two minimal slash commands to the command palette, such as /checkpoints and /restore-checkpoint <id> (or a tight equivalent naming).
2. In crates/imp-tui/src/app.rs implement the command handling by calling the session/runtime helpers from child 2.
3. Surface compact textual output in the transcript: list available checkpoints with ids/labels/files count; restore should confirm success/failure and list restored files.
4. Keep UX intentionally minimal and consistent with existing /memory and /compact command behavior.
5. Add focused tests for command palette inclusion and basic command handling behavior.

Files:
- crates/imp-tui/src/views/command_palette.rs (modify — add commands)
- crates/imp-tui/src/app.rs (modify — implement command handling)
