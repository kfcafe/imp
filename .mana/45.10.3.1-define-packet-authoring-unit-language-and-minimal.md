Goal: define the future durable authoring object/unit that specifies how a scoped subagent task packet is assembled, and define the minimal schema for both the authoring unit and the emitted packet.

Current state:
- `45.10.3` establishes that future subagent handoff should use a scoped task packet assembled on demand from delegation intent + graph slice + policy.
- User clarified a further requirement: there must be an explicit unit/object containing the instructions that create the packet. Packet construction should not be only an implicit retrieval side effect.
- The architecture still needs exact language for this authoring unit and a compact schema boundary between: (a) the authoring/unit spec and (b) the emitted scoped packet consumed by the worker runtime.

Explore and document:
1. Exact unit language for the packet-authoring object: purpose, current state, steps, scope boundaries, and anti-goals.
2. The boundary between the source authoring unit and the emitted scoped task packet.
3. Minimal required fields for the authoring unit.
4. Minimal required fields for the emitted scoped task packet.
5. Which fields are durable source-of-truth vs derived/materialized.
6. How this model preserves parent/planner ownership of decomposition and prevents worker freeform planning.
7. Compatibility expectations with current mana job-based handoff.

In scope:
- architecture wording
- schema proposal
- source-vs-derived boundary
- planner vs worker responsibility split

Out of scope:
- implementing storage/runtime code
- changing current worker prompt assembly today
- requiring every delegated task to pre-exist as a durable task record

Do not:
- treat packet assembly as raw transcript replay
- let the worker infer scope from the whole graph
- collapse the authoring unit and emitted packet into one indistinguishable object
