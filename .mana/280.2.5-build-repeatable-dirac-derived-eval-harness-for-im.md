Goal: Convert the Dirac comparison from qualitative claims into repeatable imp eval evidence.

Current state:
- Dirac's `evals/README.md` describes public complex refactor tasks and expected patch outputs, not a fully executable generic harness for imp.
- Parent unit 280.1 tried to run evals but is blocked on Gemini secret storage/keychain issue 266.4.3.4.
- imp has usage reporting work and mana verify gates, but not a Dirac-comparison-specific harness artifact yet.

Steps:
1. After auth blocker 266.4.3.4 is resolved, inspect the Dirac temp checkout or reclone `https://github.com/dirac-run/dirac` read-only.
2. Create a local harness directory such as `evals/dirac-comparison/`.
3. Add a README/spec documenting:
   - source Dirac eval commit/date.
   - each eval task name and upstream repo/commit if available.
   - setup/reset command.
   - imp command to run.
   - verifier command.
   - result schema.
4. Implement the smallest runnable script if appropriate, preferably dry-run friendly:
   - creates isolated temp checkout.
   - runs imp with provider/model parameters.
   - captures stdout/stderr transcript without secrets.
   - runs verifier.
   - stores diff and result JSON/Markdown.
5. Prove with one smallest eval first; do not attempt all evals until the one-task path works.
6. Record results back into mana 280.1.

Files:
- `evals/dirac-comparison/README.md` (create)
- `evals/dirac-comparison/run-one.sh` or similar (create if script is useful)
- `evals/dirac-comparison/results/` (generated artifacts; decide whether ignored or committed)

In scope:
- Harness/spec and one-task smoke path.
- Capturing evidence for imp vs Dirac.

Out of scope:
- Improving imp tools.
- Running expensive full benchmark matrix before one eval works.
- Printing secrets.

Do not:
- Depend on the temp Dirac clone path permanently.
- Commit API keys or provider logs that include secrets.
- Claim benchmark wins without verifier evidence.
