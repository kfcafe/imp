Write `docs/rebuild/imp-detached-execution-model.md` defining the first coherent detached/background local execution model for CLI-first `imp`.

Ground it in:
- `imp/IMP_REVIEW.md`
- `docs/rebuild/imp-cli-affordance-sequence.md`
- `docs/rebuild/imp-view-submodes-and-entrypoints.md`
- `docs/rebuild/imp-session-browse-and-search-surface.md` once created
- current session persistence/runtime docs and any existing local background state references in the repo

The doc should:
1. define the operator story for starting, inspecting, and reattaching detached/background local work
2. explain why detached/background execution depends on output contracts, browse/search surfaces, checkpoints, visible planning, and approval policy
3. identify the minimum shared runtime state required: run identity, lifecycle state, log/event persistence, reattach semantics, and policy behavior while detached
4. specify which inspection tasks belong in shell output vs `imp view` vs machine-readable status output
5. remain local-first and avoid turning `imp` into a team/cloud control plane

Acceptance criteria:
- includes the word `detached`
- includes the word `background`
- names reattach semantics
- names local-first constraints
