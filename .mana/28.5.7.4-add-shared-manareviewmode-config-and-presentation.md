## Task
Add the shared imp-core presentation layer for between-turn mana review so TUI and CLI/RPC can consume the same canonical TurnManaDelta semantics.

## Goal
Create the reusable formatter/config seam that sits between 28.5.6 runtime packet assembly and the TUI/CLI adapters.

## Grounded context
- `docs/rebuild/imp-between-turn-mana-review-ui.md` defines three user control modes: `off`, `compact_on_change`, and `pinned`.
- `docs/rebuild/mana-turn-delta-contract.md` defines the canonical semantic input: `TurnManaDelta` with explicit `no_change`, `changed`, and `needs_decision` plus `anchor_unit`, `touched_units`, `proposed_children`, `notes_appended`, `decision_events`, and `unresolved_consequential_choices`.
- `imp/crates/imp-core/src/config.rs` already owns UI display configuration and is the right place for a shared mode enum/config field.
- `imp/crates/imp-tui/src/app.rs` and `imp/crates/imp-cli/src/main.rs` both already consume shared UI abstractions, so formatting should not be reimplemented ad hoc in each adapter.

## Steps
1. Add a shared `ManaReviewMode` enum/config field in `imp/crates/imp-core/src/config.rs` with the exact modes `off`, `compact_on_change`, and `pinned`.
2. Add an imp-core review rendering module that accepts the canonical `TurnManaDelta` from 28.5.6 and returns:
   - compact line-oriented block lines for the widget/footer surface
   - expanded textual inspector lines with stable section ordering
3. Keep renderers presentation-neutral and line-oriented; they should not depend on ratatui widgets or TUI-only state.
4. Preserve explicit `no_change` / `changed` / `needs_decision` rendering.
5. Preserve separate rendering for notes, decision events, consequential unresolved choices, and dependency changes; do not collapse them into generic update text.
6. Include truncation/cap behavior consistent with `docs/rebuild/imp-between-turn-mana-review-ui.md` so adapters can render the same semantics without inventing their own summaries.

## Files
- `docs/rebuild/imp-between-turn-mana-review-ui.md` (read for compact block + textual inspector rules)
- `docs/rebuild/mana-turn-delta-contract.md` (read for canonical field semantics)
- `imp/crates/imp-core/src/config.rs` (modify — add shared review mode config)
- `imp/crates/imp-core/src` (modify — add renderer module and exports near the TurnManaDelta types from 28.5.6)
- `imp/crates/imp-core/src/ui.rs` (read only unless a tiny shared enum/helper belongs here)

## Acceptance criteria
- Shared imp-core config exposes `ManaReviewMode::{Off, CompactOnChange, Pinned}` or equivalent serialized names.
- Shared imp-core renderer functions exist for compact turn review lines and textual inspector lines.
- Renderers operate on canonical `TurnManaDelta`, not raw mana tool `ToolOutput.details`.
- Output keeps note/decision/dependency semantics distinct.

## Do not
- Do not wire TUI behavior here.
- Do not wire CLI/RPC behavior here.
- Do not invent a second schema separate from `TurnManaDelta`.
- Do not require component widgets or a graph browser.
