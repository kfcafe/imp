This epic defines the execution boundary below the live imp runtime. Goal: make command execution, worktree lifecycle, sandbox policy, and artifact capture explicit rather than split informally between runtime and CLI helpers.

Focus:
- runner request/response protocol
- local runner MVP
- worktree lifecycle
- sandbox profile model
- PTY/shell mediation
- execution artifact capture

Important decisions to resolve explicitly:
1. should the first runner implementation live inside imp or in a Tower sibling layer?
2. what minimum runner protocol is enough to replace shell-string orchestration?
3. what should be a durable lease in mana vs a transient runner concern?
4. what sandbox profiles are required for edit, verify, reviewer, and hostile execution?

Suggested child jobs:
- design the local runner protocol and payload shapes
- model worktree lease metadata and cleanup expectations
- model sandbox specs and runner-enforced execution policy
- define artifact binding for command/tool runs (`run_id`, `worktree_id`, `sandbox_id`)
- map the current `mana-cli/src/spawner.rs` behavior onto the target boundary

Done when:
- another agent could begin implementing a local runner and worktree/sandbox lease model without inventing the contract mid-stream
