## Task
Write a root architecture doc that defines the vNext mana→imp subagent handoff packet / worker-assignment shape for object-native memory.

## Why this unit exists
Current repo state already has a canonical worker seam:
- `imp/crates/imp-core/src/mana_worker.rs` loads a `WorkerAssignment` from mana and normalizes it into `TaskContext`
- `imp/crates/imp-core/src/system_prompt.rs` remains the final prompt assembler
- `mana/crates/mana-core/src/prompt.rs` is a legacy compatibility prompt-shaped bundle and should keep narrowing away from final prompt authority

The current seam is still unit-centric. As object-native memory lands, we need a precise contract for what mana should hand to imp/subagents without turning mana into the prompt renderer.

## What to specify
1. The handoff packet should remain presentation-neutral and runtime-neutral
2. It should preserve the current ownership split:
   - mana owns durable structured inheritance
   - imp owns final prompt wording, token budgeting, and live runtime behavior
3. It should define a vNext field set for subagent inheritance, likely including:
   - assignment identity and task contract
   - latest semantic checkpoint summary
   - compact bootstrap/current-state summary
   - relevant evidence refs / evidence bundles
   - accepted/current claim summaries or precursor summaries if claims are not ready yet
   - promoted lessons / runbooks
   - receipt summaries for side-effect safety
   - dependency status and open decisions
   - fallback unit-native notes/attempt summaries when object-native memory is absent
4. It should describe fallback behavior for mixed-state repos during migration
5. It should explain how this packet relates to the current `tower-contracts::worker::WorkerAssignment` and `imp` `TaskContext` types
6. It should explicitly state that this work is a follow-on architecture path and not a blocker for the current 81.5 -> 81.8 native imp delegation MVP
