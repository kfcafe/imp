Goal: define the migration and compatibility behavior from current XDG/macOS/legacy imp paths into the new canonical imp-native root contract.

Current accepted direction:
- canonical global root is `~/.imp/`
- current installs may have data spread across `~/.config/imp`, `~/.local/share/imp`, `~/Library/Application Support/imp`, and legacy `~/.imp`
- we need deterministic rules for discovery, migration, conflict resolution, and when old roots stop being authoritative

Scope of this job:
1. Inventory the legacy/non-canonical roots that may contain user-global imp files today.
2. For each surface, define the migration source candidates and canonical destination under `~/.imp/`.
3. Define conflict-resolution rules when the same logical artifact exists in multiple old roots.
4. Define whether migration is automatic copy, move, lazy-on-read reconcile, or explicit command-assisted.
5. Define operator-visible behavior and safety requirements so no existing sessions/config are stranded.

Relevant surfaces include:
- config/auth metadata
- sessions
- session indexes
- global soul/memory/user docs
- skills/prompts/tools/lua/imports

Done when:
- migration behavior is explicit enough to implement safely
- compatibility lookup roots are clearly distinguished from canonical roots
- conflict handling is specified instead of implicit

Out of scope:
- implementing the migration
- broad runtime refactors unrelated to storage
