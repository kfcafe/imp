Goal: make imp's existing usage accounting/reporting discoverable from the TUI instead of only from CLI/README knowledge.

Current state from repo inspection:
- `imp-cli` already has a real `usage` command family and `crates/imp-cli/src/usage_report.rs` implements reporting/export.
- `imp-tui/src/views/command_palette.rs` does not include a `usage` command.
- `imp-tui/src/app.rs` `/help` output does not mention usage reporting.
- Startup/welcome copy focuses on sessions, mana, and settings, but not local usage/accounting/reporting.
- `views/status.rs` shows token/cost data, but there is no obvious in-app affordance explaining historical usage/reporting.

Do the following:
1. Add a lightweight `/usage` discoverability command to the TUI command palette and slash command handling.
2. Make `/usage` return a concise user-facing explanation of what imp usage reporting is, where it comes from, and how to use the CLI command family (`imp usage ...`) today.
3. Update `/help` copy to mention usage reporting in plain English.
4. Update startup/welcome/discoverability copy to mention local usage/cost visibility if space permits without bloating the screen.
5. Keep scope tight: no new analytics backend, no full TUI usage browser in this unit.
