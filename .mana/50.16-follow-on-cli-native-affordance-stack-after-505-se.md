## Purpose
Externalize the durable follow-on decomposition from `docs/rebuild/imp-cli-affordance-sequence.md` into root mana.

## Ordered slices
1. stable human vs machine output modes
2. checkpoints
3. visible planning
4. approval policy
5. session browsing/search
6. detached/background execution

## Dependency intent
- output modes are the first shared-runtime prerequisite
- checkpoints are mixed runtime+surface work and should follow output-mode stabilization
- visible planning follows checkpoints so plan/review/apply sits on top of legible rewind
- approval policy follows visible planning so approval attaches to visible intent instead of arbitrary prompts
- session browsing/search is mostly surface-level and can overlap late trust-stack work, but remains sequenced after the earlier contract-setting slices in this decomposition
- detached/background execution comes last because it depends on output contracts, checkpoints, approval states, and reattach/browse surfaces
