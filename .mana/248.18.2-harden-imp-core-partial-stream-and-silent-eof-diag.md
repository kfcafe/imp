Goal: improve robustness and user-facing diagnostics when provider streams fail after partial output or end without a terminal completion event.

Current state:
- agent.rs already surfaces partial-stream failures and silent EOF as AgentEvent::Error.
- The existing error strings are technical and not clearly classified for downstream surfaces.
- Tests exist for partial stream failure and silent EOF, but there is room to make wording and coverage more explicit.

Steps:
1. Inspect agent.rs stream loop, retry.rs startup retry behavior, and current tests for partial-stream and silent-EOF cases.
2. Tighten error wording so downstream surfaces can distinguish at least: provider-declared stream error, transport/stream failure after partial output, and premature stream termination without MessageEnd.
3. Preserve current correctness guarantees: no silent success synthesis after missing MessageEnd, no retry after meaningful streamed output has already been emitted.
4. Expand focused tests in imp-core to cover the final surfaced messages and verify event ordering around partial output + error.

Files:
- imp/crates/imp-core/src/agent.rs
- imp/crates/imp-core/src/retry.rs

In scope:
- Stream termination diagnostics
- Existing retry boundary guarantees
- Focused imp-core tests

Out of scope:
- Cross-surface rendering polish beyond what imp-core exposes
- JSON/RPC envelope redesign
