Goal: Add Dirac-style stale-safe anchored editing by improving existing `read` and `edit`, without adding separate `anchor_read` or `anchor_edit` tools.

Current state:
- User clarified: anchors should be part of `read`; focus on `read`, `scan`, and `edit` only.
- `crates/imp-core/src/tools/read.rs` reads files and returns line-oriented output.
- `crates/imp-core/src/tools/edit.rs` edits by exact/fuzzy `oldText`, supports dryRun, expectedOccurrences, replaceAll, and delegates array edits to `multi_edit`.
- `ToolContext` includes `file_tracker`, `file_cache`, and `checkpoint_state`.
- Current stale behavior warns on unread/stale files but does not provide opaque anchors or anchor-range editing.

Steps:
1. Depend on 280.2.1 contract; use the chosen schema from that doc.
2. Extend `read` with an opt-in parameter such as `anchors: true` that returns opaque, file-scoped line anchors alongside normal line-oriented output.
3. Generate opaque, file-scoped, per-line anchors from stable data such as file hash + line number + line content hash. Anchors are not security tokens.
4. Persist enough per-session anchor state in `ToolContext` or a small shared state type to validate that an anchor still matches current file content before editing.
5. Extend `edit` with an anchored replacement mode in addition to the existing exact/fuzzy text mode:
   - path required.
   - start/end anchor or anchor range required.
   - replacement text required.
   - dryRun supported.
   - stale/missing anchor errors are blocking.
   - checkpoint before write.
   - returns diff and refreshed anchor metadata.
6. Preserve existing `edit` behavior and tests.
7. Add tests for: happy path replace, stale file rejection, dry-run no write, checkpoint path invoked/recorded, and existing edit still works.

Files:
- `crates/imp-core/src/tools/read.rs` (modify — opt-in anchors)
- `crates/imp-core/src/tools/edit.rs` (modify — add anchored edit mode while preserving exact mode)
- `crates/imp-core/src/tools/mod.rs` (modify — add shared anchor state to `ToolContext` if needed)
- `crates/imp-core/src/tools/multi_edit.rs` (inspect; modify only if anchored array edits are supported in this slice)

In scope:
- Anchored line/range replacement through existing tools.
- Session-local stale safety.
- Checkpoint integration.

Out of scope:
- New model-facing `anchor_*` tools.
- AST-aware symbol replacement.
- Multi-file transaction batching beyond existing `multi_edit` unless trivial.
- IDE diagnostics.

Do not:
- Remove fuzzy/exact edit support.
- Store large full-file anchor maps in durable session files in v1 unless contract explicitly calls for it.
- Treat anchors as security boundaries.
