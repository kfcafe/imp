Goal: Bring Dirac's batch edit discipline into imp by improving existing `edit`/`multi_edit`: validate, preview, checkpoint, and apply multiple edits as one operation.

Current state:
- User clarified: focus on improving `read`, `scan`, and `edit`; avoid separate new tools.
- `crates/imp-core/src/tools/edit.rs` delegates `edits` arrays to `multi_edit`.
- Existing `multi_edit` likely applies sequential exact/fuzzy edits, but the Dirac comparison found imp lacks a first-class transaction concept with combined validation/evidence.
- Checkpoint support exists through `ToolContext.checkpoint_state`.

Steps:
1. Inspect existing `crates/imp-core/src/tools/multi_edit.rs` before changing anything.
2. Align with 280.2.1/280.2.3 contracts.
3. Implement transaction semantics inside the existing `edit` array/multi-edit path rather than adding a separate `edit_transaction` tool:
   - group edits by file.
   - validate all target ranges/text/anchors before writing any file.
   - reject overlapping edits within a file.
   - produce one combined diff in dry-run and write modes.
   - checkpoint all touched files before writes.
   - invalidate file cache and refresh tracking after writes.
   - report structured `details`: files, edit_count, checkpoint_id if available, dry_run, rejected overlaps/stale anchors.
4. If using anchors, depend on 280.2.3. If using exact text first, document that anchor integration is follow-up.
5. Add tests for:
   - multiple non-overlapping edits in one file.
   - edits across two files.
   - overlapping edits rejected with no writes.
   - dry run writes nothing.
   - checkpoint created before write.

Files:
- `crates/imp-core/src/tools/multi_edit.rs` (modify or reuse)
- `crates/imp-core/src/tools/edit.rs` (modify only if needed to delegate transaction mode)
- `crates/imp-core/src/tools/mod.rs` (modify only if shared state/reporting changes)

In scope:
- Existing edit/multi_edit transaction behavior.
- Combined diff/evidence.

Out of scope:
- New model-facing `edit_transaction` tool.
- Automatic model-turn batching inside the agent loop.
- Semantic AST rewrite tools.
- IDE approval UI.

Do not:
- Apply partial writes after validation failure.
- Hide overlap/stale failures behind fuzzy matching.
- Break current `edit` and `multi_edit` callers.
