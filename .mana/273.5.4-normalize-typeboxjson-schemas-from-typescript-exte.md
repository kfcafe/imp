Goal: ensure schemas registered by Pi-style TypeScript extensions become provider-compatible imp tool schemas.

Context:
- Existing Pi extensions commonly use `@sinclair/typebox` `Type.*` as `parameters`.
- imp has provider schema restrictions and existing work around top-level JSON Schema compatibility.

Steps:
1. Inspect what TypeBox emits at runtime for representative Pi extension parameters.
2. Add normalization for common schema shapes used by current extensions.
3. Reject or simplify provider-incompatible schema features with actionable errors.
4. Preserve tool names/descriptions and parameter descriptions where possible.
5. Add fixture tests for string/boolean/enum/union/object/array shapes used by audited Pi extensions.

Likely related files:
- `imp/crates/imp-core/src/tools/*`
- TS runtime adapter module from sibling task
- existing schema tests around imp tool definitions

Out of scope:
- Full JSON Schema support.
- Silent lossy conversion that changes semantics.
