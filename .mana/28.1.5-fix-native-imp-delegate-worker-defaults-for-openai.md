## Goal
Make native `imp` worker delegation behave correctly when the operator or caller relies on configured defaults or OAuth-backed OpenAI auth.

## Current verified state
- A native `imp` tool delegation smoke test against imp-local mana unit `46.1` reached the worker runtime.
- Forcing `provider="openai"` with OAuth-only auth fails because the provider auto-switch to `openai-codex` only happens when provider selection is left to normal resolution.
- The `imp` tool currently treats optional string params like `system_prompt`, `model`, and `provider` as present even when they are the empty string.
- In `crates/imp-core/src/tools/imp.rs`, both unit and ad-hoc delegation paths map empty `system_prompt` to `Some("")`.
- In `crates/imp-core/src/builder.rs`, any `Some(system_prompt)` overrides assembled prompt generation entirely.
- With OpenAI OAuth / `openai-codex`, that empty override produces an empty instructions field and the provider call fails with `HTTP 400 Bad Request: {"detail":"Instructions are required"}`.

## Scope
- In scope:
  - normalize optional delegate params so empty strings are treated as absent
  - preserve auto provider resolution for OAuth-backed OpenAI/Codex flows
  - add regression coverage for the native `imp` tool delegate path and worker session creation path
- Out of scope:
  - redesigning broader auth UX
  - changing provider selection semantics outside the bug fix needed here

## Concrete tasks
1. Update native `imp` tool delegate param parsing so optional string fields ignore empty-string values instead of converting them to `Some("")`.
2. Confirm unit delegation and ad-hoc delegation both preserve default model/provider/system prompt behavior when those fields are omitted or blank.
3. Add regression coverage showing an OAuth-only OpenAI setup can still reach the ChatGPT/Codex path when provider is not forced.
4. Add regression coverage for blank `system_prompt` not clobbering the assembled prompt in delegate-driven worker runs.
5. Re-run a native `imp` delegate smoke test against a safe unit after the fix.
