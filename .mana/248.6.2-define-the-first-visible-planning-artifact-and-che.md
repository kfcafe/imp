Goal: define the first visible planning artifact/checklist flow for the flagship shell, with matching semantics for the TUI cockpit later.

Current state:
- Prior review and current audit both identify visible planning as a major missing trust surface.
- The premium shell spec explicitly calls for plans/checklists to become first-class artifacts rather than hidden reasoning.
- The next step is to decide the smallest honest plan surface that can be shown in shell/TUI without promising a full planner product all at once.

Steps:
1. Inspect current planning-adjacent behavior and existing docs/spec language.
2. Define what the first visible plan artifact contains: intended actions, likely files/surfaces, risks, verify/check step, and next action.
3. Decide where the artifact appears in shell, where it hands off to viewer/cockpit, and how it remains honest about maturity.
4. Create implementation follow-up only after the contract is explicit.
