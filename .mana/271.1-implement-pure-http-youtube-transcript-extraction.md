Goal: replace the current YouTube rejection in `crates/imp-core/src/tools/web/read.rs` with a pure Rust/HTTP extraction path for YouTube watch URLs. First slice: no yt-dlp, no video/audio download, no new external service. Steps: 1) Add a focused YouTube module under `crates/imp-core/src/tools/web/` that parses video IDs from youtube.com/watch, youtu.be, shorts, embed URLs. 2) Fetch the watch page with existing reqwest client/browser-like headers. 3) Extract `ytInitialPlayerResponse` from HTML using balanced JSON extraction, not brittle regex-only end markers. 4) Parse `videoDetails`, `microformat.playerMicroformatRenderer`, and `captions.playerCaptionsTracklistRenderer.captionTracks`. 5) Select preferred transcript track in order: manually provided English, English original/auto, first non-translatable track. 6) Fetch caption `baseUrl` with `fmt=json3` when needed; parse JSON3 `events[].segs[].utf8`, preserving rough timestamps. 7) Return a `PageContent` with title, URL, status, content type, diagnostics, and `<web_content>` body generated by existing web formatting. 8) Add unit tests for video ID parsing, balanced JSON extraction, transcript JSON3 parsing, and track selection using fixtures/inline JSON. Scope boundaries: do not shell out to yt-dlp; do not download media; do not add a heavy dependency without asking; do not implement audio transcription yet. Verify: `cargo test -p imp-core tools::web::youtube tools::web::read::tests::youtube` or closest targeted package tests.
