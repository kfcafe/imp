Implement the first pure Rust/HTTP YouTube ingestion slice in imp. Current state: `imp/crates/imp-core/src/tools/web/read.rs` detects YouTube URLs and returns `ReadError::YoutubeNotSupported`. Desired state: that branch calls a native YouTube extractor and returns `PageContent` compatible with existing `web.read` formatting. Concrete steps: 1) Add `imp/crates/imp-core/src/tools/web/youtube.rs`. 2) Implement robust video ID parsing for `youtube.com/watch?v=...`, `youtu.be/...`, `/shorts/...`, and `/embed/...`. 3) Fetch the watch page with the existing reqwest client and browser-like headers already used by web read. 4) Extract `ytInitialPlayerResponse` from HTML via balanced-brace JSON scanning anchored at the assignment, avoiding a brittle `;</script>`-only parser. 5) Parse video metadata from `videoDetails` and `microformat.playerMicroformatRenderer` where available: title, channel/uploader, duration, upload/publish date, view count if present, description if safely available, and canonical URL. 6) Parse `captions.playerCaptionsTracklistRenderer.captionTracks`; select track deterministically: manual English first, English original/auto second, otherwise first usable track. 7) Fetch selected caption `baseUrl` with `fmt=json3` when the URL does not already specify it; parse `events[].segs[].utf8` into timestamped transcript lines while skipping empty/music-only segments where obvious. 8) Return `PageContent` with content type/format diagnostics and web-safe body including source attribution, metadata, caption track diagnostics, chapters if available from `playerOverlays`/initial data only if straightforward, and transcript. Scope boundaries: do not shell out to yt-dlp; do not download video/audio; do not add a heavy dependency without explicit approval; do not implement audio transcription/frame sampling in this unit; keep changes focused to imp-core web read/extractor tests unless a small module export is necessary.
