  - `imp run <unit-id>` — canonical single-unit worker/runtime path
  - `imp view` — viewer/inspector for logs, sessions, branches, checkpoints, tool activity, and agent/session browsing
  - `imp tui` — optional richer fullscreen adapter/control-room surface
  - retain one-shot path (`imp -p` today; evaluate `imp prompt` as clearer long-form command while preserving `-p` as shorthand)

  Direct CLI subcommands to expose shared capabilities:
  - `imp settings`
  - `imp personality`
  - `imp login`
  - `imp secrets`
  - `imp models`
  - `imp sessions`
  - `imp memory`
  - `imp checkpoints`
  - existing `imp usage`, `imp config`, `imp import`, `imp web-login` remain relevant

  Canonical CLI-shell namespace (prefer colon syntax):
  1. Session/state
     - `:new`
     - `:resume [query]`
     - `:fork`
     - `:name <name>`
     - `:export [path]`
     - `:tree`
     - `:history` (candidate if tree/history should split)
  2. Runtime controls
     - `:model [name]`
     - `:thinking [off|minimal|low|medium|high|xhigh]`
     - `:compact`
     - `:reload`
     - `:status`
     - `:quit`
  3. Setup/config/identity
     - `:settings [field] [value]`
     - `:personality [show|edit|set ...]`
     - `:login [provider]`
     - `:secrets [provider]`
     - `:setup`
  4. Memory/checkpoints
     - `:memory ...`
     - `:checkpoints`
     - `:restore-checkpoint <id-or-label>`
  5. Utility/file/help
     - `:help [topic]`
     - `:copy [last|turn <n>|selection]`
     - `:files [query]`
     - preserve useful `@file` inclusion behavior with tab-completion or CLI picker support where possible
  6. Viewer/TUI handoff
     - `:view [logs|tree|session|checkpoints|tools]`
     - `:tui [settings|personality|session|logs]`
  7. Extension commands
     - remain in same namespace, discoverable via `:commands`
     - extension command list should distinguish built-ins vs extensions and show help/ownership

  Recommended migration behavior:
  - Phase 1: add `imp chat`, `imp view`, and `imp tui` explicitly while keeping current default behavior stable
  - Phase 2: once shell is strong enough, flip plain `imp` to the CLI shell and keep `imp tui` as explicit fullscreen entrypoint
  - During migration, accept legacy `/command` in the shell but treat `:command` as canonical

  Important shell UX details to preserve functionality beyond literal commands:
  - lightweight prompt status row (project/model/thinking/session/unit)
  - line editing + multiline input
  - streaming output with compact tool notices
  - inline ask/confirm flows
  - transcript summaries after turns
  - no fullscreen dependence for key operator actions
