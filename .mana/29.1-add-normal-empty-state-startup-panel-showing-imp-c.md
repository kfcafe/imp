Goal: when the normal TUI opens with no chat history, render a startup panel in the main chat area that makes imp's available capabilities discoverable without requiring the welcome flow or README.

Current state:
- First-run welcome flow in `crates/imp-tui/src/views/welcome.rs` shows setup content, but normal startup with empty history renders no capability overview.
- `crates/imp-tui/src/app.rs` always renders the chat area from message lines; when there are no messages it stays visually empty.
- imp already has the data needed to build this surface:
  - built-in / mode-allowed tools from `crates/imp-core/src/builder.rs` and `crates/imp-core/src/tools/mod.rs`
  - discovered skills from `crates/imp-core/src/resources.rs`
  - loaded Lua extensions, commands, and tools via `crates/imp-lua/src/loader.rs`, `crates/imp-lua/src/sandbox.rs`, and `App::reload_lua_extensions()` in `crates/imp-tui/src/app.rs`
  - assembled system prompt can be built through the same `AgentBuilder` path already used in `app.rs`

Desired UX shape:
- Only show this panel in the normal empty startup state (no messages, not first-run welcome overlay).
- Prefer a multi-column layout with compact sections such as tools, skills, extensions/commands, and session/model/auth/cwd.
- Include system prompt visibility in some form, but avoid a giant wall of text that overwhelms the overview. Treat the prompt as a preview/toggle/detail area rather than the primary content unless a later decision says otherwise.

Implementation outline:
1. Add a dedicated empty-state/startup view under `crates/imp-tui/src/views/` that can render several compact columns plus an optional bounded system prompt preview.
2. Add app-side startup-state assembly in `crates/imp-tui/src/app.rs` to gather:
   - effective tools available in the current mode
   - discovered skills for the current cwd
   - loaded Lua extension tools and slash commands from `self.lua_runtime`
   - current model/provider/auth/session/cwd summary
   - assembled system prompt text for previewing
3. Update normal render flow in `crates/imp-tui/src/app.rs` so when `UiMode::Normal` and `self.messages.is_empty()`, the chat area renders the startup panel instead of an empty transcript.
4. Keep scope tight:
   - do not change first-run welcome flow behavior in this unit
   - do not add new backend capabilities or new extension systems
   - do not claim TypeScript extensions are shipped; current extension reality is Lua
   - do not make the prompt preview unbounded by default
5. Add focused tests for the empty-state rendering/data shaping where practical, and verify with `cargo check -p imp-tui` plus any targeted tests added.
