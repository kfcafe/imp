Goal: collect and act on early operator feedback from the new default interactive CLI shell without losing the findings in chat transcripts.

Current state:
- bare `imp` now launches the CLI shell by default via 248.8
- user wants to start using it and provide feedback immediately
- the shell is good enough to start the feedback loop, but the next improvements should be driven by real operator friction rather than abstract guessing

Steps:
1. Define the feedback categories to capture durably: startup/entrypoint feel, status line usefulness, command discovery, shell rhythm, turn summaries, shell↔view↔tui handoff quality, and reasons the user still reaches for the cockpit.
2. Create a simple way to append concrete observations from live use into mana on this unit or child notes.
3. Decompose likely follow-on slices once repeated feedback patterns emerge.

Files:
- imp/crates/imp-cli/src/main.rs
- imp/README.md
- docs/design/imp-premium-interactive-cli.md

Deliverable:
- durable feedback loop notes and explicit follow-on jobs as patterns emerge

Do not:
- treat transient chat complaints as durable product direction until they repeat or are clearly consequential
- hide feedback only in session transcripts
