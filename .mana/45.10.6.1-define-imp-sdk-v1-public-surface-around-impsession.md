Goal: define the minimum stable public API for imp SDK v1 using the current proto-SDK pieces already in `imp-core`.
Current state: `ImpSession`, `SessionOptions`, `SessionChoice`, `AgentEvent`, and `UserInterface` already exist, but they are not yet framed as a coherent supported host API.

Decide and document:
1. Which current types/methods become the SDK v1 surface.
2. Which current accessors/events are stable vs provisional/internal.
3. The minimum host capabilities required in v1: session creation, prompt lifecycle, control commands, event streaming, model/thinking changes, persistence policy, UI bridge.
4. What is explicitly out of scope for SDK v1.

Files/context:
- `imp/crates/imp-core/src/imp_session.rs`
- `imp/crates/imp-core/src/agent.rs`
- `imp/crates/imp-core/src/ui.rs`
- `imp/crates/imp-core/src/lib.rs`
- `~/pi-mono/packages/coding-agent/docs/sdk.md` (benchmark only)

Deliverable:
- a concise memo or mana note naming the stable SDK v1 public surface.

Do not:
- propose a full rewrite
- expose CLI/TUI-specific orchestration as part of the SDK contract
