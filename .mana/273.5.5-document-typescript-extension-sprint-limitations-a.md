Goal: document what the sprint supports so users understand import vs execution and which Pi extensions are expected to work.

Content to capture:
- Storage layout: `.imp/extensions/` and `~/.config/imp/extensions/pi/`.
- Import command: `imp import --from pi` imports skills, AGENTS.md, and TypeScript extensions.
- Supported runtime subset: Pi-style `registerTool` first; commands/hooks/UI/deps may be partial.
- Known-good low-dependency targets such as `ask.ts`, `color-palette`, `youtube`, or a fixture extension.
- Known deferred/complex targets: package-local dependency-heavy extensions such as `web`, `database`, `vibecheck` unless runtime dependency loading is implemented.
- Lua remains supported and unchanged.

Likely files:
- `imp/README.md`
- `imp/docs/proposals/guest-runtime-extension-substrate.md` or a sprint note under docs/proposals

Out of scope:
- Overstating TypeScript API parity.
- Describing TS extension support as complete if only the minimal subset works.
