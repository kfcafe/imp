## Goal
Plan the first proving slice for read-oriented code intelligence in `imp`: semantic diagnostics plus AST-backed structural alignment.

## Current state
The read-oriented plan recommends diagnostics as the first semantic priority and explicitly keeps AST structural inspection as a first-class companion. This slice should answer 'what is broken and where?' with compact output and nearby structural context.

## Scope
- Planning only.
- Cover semantic diagnostics, AST fallback/context extraction, and mapping into a shared problem-list shape.
- Include transcript/viewer behavior for compact problem summaries.

## Acceptance
- The slice explains how semantic diagnostics and AST structure compose rather than duplicate.
- It identifies the minimal runtime outputs needed to summarize blockers by file/location while exposing structural nearby context.
- It records how this slice should relate to compiler/linter/build/test outputs without collapsing them into a single backend protocol.

## Important constraints
- Do not assume semantic diagnostics replace deterministic compiler/linter/test truth.
- Do not require write-oriented semantic actions.
- Keep degraded/fallback answers explicit when the semantic backend is unavailable or stale.
