Goal: define and, if chosen, implement a stable machine-facing error envelope for streamed imp output without regressing existing consumers.

Current state:
- imp-cli JSON and RPC output emit flat string fields for error/stream_error events.
- Human-facing surfaces need normalization, but machine-facing consumers may need both concise classification and raw detail.

Steps:
1. Audit current JSON and RPC event shapes in imp-cli/src/lib.rs.
2. Decide whether streamed errors need structured fields such as class/kind, display_message, raw_message, retryable, and stream_phase.
3. Preserve backward compatibility or document the exact compatibility break if schema changes are necessary.
4. Add/adjust tests only if this slice is explicitly selected.

Files:
- imp/crates/imp-cli/src/lib.rs
- imp/crates/imp-llm/src/stream.rs
- imp/crates/imp-core/src/agent.rs

In scope:
- Machine-facing contract design
- Compatibility assessment

Out of scope:
- Human-only rendering polish already covered by sibling jobs
