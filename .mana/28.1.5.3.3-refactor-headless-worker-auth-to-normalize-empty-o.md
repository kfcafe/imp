Goal: make headless worker auth clean and predictable by normalizing empty API-key overrides to absent values, resolving credentials through one precedence order, and failing clearly instead of sending empty tokens.

Current state:
- Empty or missing auth in the no-tools/headless path can collapse into an empty string token.
- The worker/session path special-cases OpenAI auth in ways that bypass saved OAuth.
- The user wants auth resolution to check saved auth first, then API-key sources.

Steps:
1. Normalize empty/whitespace API-key overrides to `None` at the CLI/session boundary.
2. Refactor runtime auth resolution so it uses one precedence order for the chosen route: explicit non-empty override -> saved auth -> stored API key -> env -> clear error.
3. Remove any auth-failure-to-empty-string fallback from headless/no-tools worker startup.
4. Ensure OpenAI/ChatGPT saved auth is considered where the chosen route can use it.
5. Add focused tests for empty-string normalization, saved-auth preference, and explicit failure behavior.

Files:
- `crates/imp-cli/src/lib.rs`
- `crates/imp-core/src/imp_session.rs`
- `crates/imp-llm/src/auth.rs`

In scope:
- auth precedence and normalization for headless workers
- explicit error behavior instead of empty token fallthrough

Out of scope:
- broader credential-store redesign
