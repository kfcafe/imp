Goal: improve the normal empty-state startup panel so it behaves like a launch surface, not just an inventory screen.

Current state:
- `crates/imp-tui/src/views/startup.rs` renders a multi-section overview of tools, skills, extensions, session metadata, and a generated prompt preview.
- `crates/imp-tui/src/app.rs` shows that panel only when `UiMode::Normal` and there are no chat messages.
- The current panel is informative, but the first thing a user sees is mostly passive inventory rather than obvious next actions.

Steps:
1. Update the startup panel data model and rendering so the top of the panel includes a compact, prominent quick-start/action area.
2. Add keyboard-oriented launch actions that match existing TUI behavior rather than introducing new backend systems. Likely candidates: resume session, open settings, open welcome/setup, open model picker, and show help/commands.
3. Keep the actions understandable to a non-developer user: the screen should answer "what can I do right now?" before listing runtime surface details.
4. Preserve the current empty-state scope only; do not change the first-run welcome flow in this unit.
5. Verify the new layout still works across narrow and wide terminal widths without crowding out the rest of the startup surface.

Files:
- `crates/imp-tui/src/views/startup.rs` (modify — add action-oriented launch area and responsive layout)
- `crates/imp-tui/src/app.rs` (modify — assemble startup actions and any labels/state needed for rendering)
- nearby TUI view files only as reference for existing shortcut patterns

In scope:
- launch-oriented startup copy
- discoverable quick actions tied to existing flows
- layout adjustments needed to support the action area

Out of scope:
- new backend capabilities
- changing first-run setup flow semantics
- large command-system refactors

Do not:
- turn the startup panel into a verbose help page
- add fake or dead-end actions that do not map to existing behavior
