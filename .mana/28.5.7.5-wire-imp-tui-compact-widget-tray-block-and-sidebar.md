## Task
Use the canonical TurnManaDelta and shared rendering/config seams to add the first shippable TUI review surface: a compact widget tray block plus a sidebar/detail textual inspector.

## Goal
Replace raw per-action mana widget noise with a turn-batched between-turn review surface in the TUI.

## Grounded context
- `imp/crates/imp-tui/src/app.rs` currently renders the widget tray as a single bordered paragraph titled `widgets`, with fixed `widget_height = 5` whenever any widget exists.
- The tray prepends the widget key (`mana`) as its own line, so the current fixed height leaves too little room for a useful compact block.
- `WidgetContent::Component` is placeholder-only in the TUI, so first ship must remain line-oriented.
- `imp/crates/imp-tui/src/views/sidebar.rs` already has the line-based detail path needed for a lightweight textual inspector.
- `docs/rebuild/imp-between-turn-mana-review-ui.md` defines the first ship as a widget tray compact block plus a sidebar/detail textual inspector, with default mode `compact_on_change` and question-gating only for consequential unresolved choices.

## Steps
1. Read the shared `ManaReviewMode` config and the shared compact/textual renderer outputs from the dependency units.
2. Update `imp/crates/imp-tui/src/app.rs` so between-turn mana review uses the frozen turn packet instead of relying on raw per-action mana widget writes.
3. Make the widget tray mana-aware: when the between-turn mana review block is present, reserve enough tray height to display the compact block without relying on wrap noise.
4. Render the compact block in `WidgetContent::Lines` form, preserving explicit `no_change`, `changed`, and `needs_decision` states.
5. Reuse the existing sidebar/detail line rendering path to show the expanded textual inspector; do not introduce a graph browser or component widget framework.
6. Respect `ManaReviewMode` behavior:
   - `off`: no auto-show
   - `compact_on_change`: show only for `changed` / `needs_decision`
   - `pinned`: keep the block visible even for `no_change`
7. Keep question-gating separate: the TUI should surface the review state, but only consequential unresolved choices should trigger the ask flow.

## Files
- `docs/rebuild/imp-between-turn-mana-review-ui.md` (read)
- `imp/crates/imp-tui/src/app.rs` (modify — widget tray sizing, compact block, inspector surfacing)
- `imp/crates/imp-tui/src/views/sidebar.rs` (modify only if a dedicated textual-inspector path/helper is needed)
- `imp/crates/imp-tui/src/tui_interface.rs` (read; modify only if the turn review packet needs a cleaner UI request path)
- shared imp-core TurnManaDelta + renderer/config module from 28.5.6 / sibling unit (read/use)

## Acceptance criteria
- The TUI shows a turn-batched mana review block instead of depending on per-action widget noise.
- The TUI supports `off`, `compact_on_change`, and `pinned` behavior.
- An expanded textual inspector can be viewed through existing sidebar/detail surfaces.
- The first ship remains line-oriented and does not require a graph browser.

## Do not
- Do not re-derive semantics from raw mana ToolOutput.details inside the TUI.
- Do not require Wizard.
- Do not leave the tray at a shape that only shows the key and one truncated line.
