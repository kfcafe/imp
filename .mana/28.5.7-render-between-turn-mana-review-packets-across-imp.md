## Task
Consume the turn-scoped mana review packet from the runtime and render a compact between-turn review surface in imp-first environments.

## Goal
Make the mana graph continuously reviewable between turns without requiring manual status checks or Wizard.

## Dependency
This unit depends on the imp-core review packet/accumulator unit landing first.

## Grounded context
- `docs/rebuild/mana-between-turn-review-affordance.md` specifies the first ship as a compact line-based review surface using existing imp UI abstractions.
- `imp/crates/imp-tui/src/tui_interface.rs` and `imp/crates/imp-tui/src/app.rs` already support `SetWidget`/`SetStatus` and render a compact widget tray.
- `imp/crates/imp-cli/src/main.rs` already forwards `set_status` and `set_widget` through its RPC/event path, so the same packet should be renderable outside the TUI.
- Current `WidgetContent::Component` handling in the TUI is placeholder-only, so first ship should use line widgets / compact textual rendering.

## Required behavior
1. Render a single between-turn review summary after a turn completes.
2. Support explicit no-change behavior (`mana: no durable changes this turn`) in a quiet but visible way.
3. Render changed/needs-decision states compactly using touched ids/titles, new child jobs, and unresolved decisions.
4. Ask only for consequential unresolved choices using existing ask/select/input flows when the packet says a decision is required.
5. Keep the rendering path reusable across TUI and CLI/RPC surfaces.

## Implementation approach
- Add a small renderer/formatter that turns the aggregated review packet into compact line output.
- Wire TUI widget/status updates to the end-of-turn packet rather than only to individual mana tool calls.
- Ensure line-oriented / RPC consumers can emit the same information as footer/status/widget-like events.
- Keep default output compact; do not dump raw mana tree/status output.

## Files
- `docs/rebuild/mana-between-turn-review-affordance.md` (read for UX rules)
- `imp/crates/imp-tui/src/tui_interface.rs` (read)
- `imp/crates/imp-tui/src/app.rs` (modify — show final packet in widget tray/status)
- `imp/crates/imp-cli/src/main.rs` (modify — surface the packet in CLI/RPC-friendly form)
- `imp/crates/imp-core/src/ui.rs` (read / small shared helper if needed)
- imp-core runtime packet source introduced by dependency unit (read/use)

## Acceptance criteria
- End-of-turn mana review is visible in imp without Wizard.
- No-change, changed, and needs-decision states render distinctly.
- Existing surfaces stay compact and non-spammy.
- Consequential unresolved choices can trigger a concise question instead of hidden ambiguity.

## Do not
- Do not introduce a heavy custom component framework.
- Do not require fullscreen TUI ownership of the feature.
- Do not make every mutation ask for explicit approval.
