This epic converts Tower's verification-first doctrine into imp-side runtime outputs and mana-side durable records. Goal: make candidate → evidence → gate a literal workflow shape instead of an architectural aspiration.

Focus:
- edit-stage outputs vs verify-stage outputs
- evidence bundle shape
- skeptic/review stage inputs and outputs
- completion gating from artifacts rather than narration

Important decisions to resolve explicitly:
1. what minimum evidence bundle should every non-trivial imp run emit?
2. which verification artifacts are ephemeral runtime detail vs durable mana records?
3. where should skeptic/review run in the workflow and what should it consume?
4. how should completion status encode missing checks, weak evidence, or review requirements?

Suggested child jobs:
- define the minimum coding-run evidence bundle
- separate patch-generation output from verification output in the worker path
- define skeptic/review artifact schema
- specify completion statuses and evidence-based gate rules
- map current mana verify history onto the richer evidence model

Done when:
- verification and review are broken into explicit implementation slices with clear durable outputs
