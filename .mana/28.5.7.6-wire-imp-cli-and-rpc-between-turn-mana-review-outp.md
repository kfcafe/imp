## Task
Render the canonical TurnManaDelta across imp CLI/RPC surfaces so non-TUI operators get the same between-turn mana review semantics.

## Goal
Make the first ship line-oriented and CLI-first compatible, not TUI-only.

## Grounded context
- `imp/crates/imp-cli/src/main.rs` already forwards `set_status` and `set_widget` through the RPC event stream.
- `docs/rebuild/imp-between-turn-mana-review-ui.md` requires the textual inspector presentation to remain neutral enough that CLI/RPC can mirror the same semantics.
- `docs/rebuild/mana-turn-delta-contract.md` says CLI/RPC consumers should render the same contract, not re-derive semantics from raw tool details.

## Steps
1. Consume the shared `ManaReviewMode` config and shared compact/textual renderer outputs from the dependency units.
2. Add CLI-friendly between-turn review output for the compact block semantics so ordinary chat/shell flows can show the same review state after each assistant turn.
3. Ensure RPC mode emits the same review semantics over its existing UI/event path rather than inventing a parallel payload.
4. Preserve explicit `no_change`, `changed`, and `needs_decision` output.
5. Preserve question-gating so only `unresolved_consequential_choices` can trigger a direct follow-up ask, and ask one concise question at a time.
6. Keep the output line-oriented and compact; do not dump raw mana tree/status output.

## Files
- `docs/rebuild/imp-between-turn-mana-review-ui.md` (read)
- `imp/crates/imp-cli/src/main.rs` (modify — CLI footer/event wiring)
- shared imp-core TurnManaDelta + renderer/config module from 28.5.6 / sibling unit (read/use)
- `imp/crates/imp-core/src/ui.rs` (read only unless a tiny shared UI helper is needed)

## Acceptance criteria
- CLI and RPC consumers surface the same between-turn mana review semantics as the TUI compact block.
- Output distinguishes `no_change`, `changed`, and `needs_decision`.
- Output remains compact and line-oriented.
- Direct follow-up questions are asked only for consequential unresolved choices.

## Do not
- Do not make CLI review depend on TUI-only types.
- Do not invent a CLI-only schema.
- Do not spam raw per-action mana widget strings after each tool call.
