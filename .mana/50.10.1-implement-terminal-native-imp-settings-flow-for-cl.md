Build the first guided CLI parity flow now that `imp chat` and `imp view` are real. Implement `imp settings` as a direct subcommand and `:settings` as a shell command inside `imp chat`, using a shared underlying settings flow where practical. Ground this work in the current settings capability breadth already inspected from `imp/crates/imp-tui/src/views/settings.rs`.

Scope for the first slice:
1. provide a terminal-native guided settings editor/listing flow rather than just opening config files
2. support at least a meaningful subset of current settings with typed editing behavior:
   - model
   - thinking level
   - max tokens
   - max turns
   - tool output mode / display-related setting(s)
   - web search provider if cheap from current code paths
3. expose quick usage patterns where sensible, but keep the main path interactive and guided
4. make the same capability reachable from both `imp settings` and `imp chat` via `:settings`
5. keep the flow fullscreen-optional; `imp tui` may remain a richer editor, but must not be required

Out of scope for this slice:
- full parity for every existing settings field
- personality/setup flows (separate children under 50.10)
- broad viewer work

Likely files:
- `imp/crates/imp-cli/src/main.rs`
- new extracted CLI flow/helper modules under `imp-cli` if needed
- shared config helpers if obviously warranted

Verification should stay narrow:
- `cargo check -p imp-cli`
- targeted tests for settings command parsing or flow helpers if introduced
- one or more simple smoke runs covering `imp settings` and/or `:settings` in plain terminal mode
