## Goal
Add the first high-value read-oriented semantic queries on top of the shared Phase 1 code-intelligence contract.

## Current state
- The plan doc prioritizes `diagnostics`, `go-to-definition`, and `find references` as the first semantic slice because they give the strongest navigation and correctness payoff.
- `imp` already has AST-backed `scan` for structure, but not exact workspace-aware binding or semantic diagnostics.

## Scope
- semantic diagnostics query surface
- `go-to-definition`
- `find references`
- normalized summaries and truncation behavior
- no rename/code actions

## Acceptance
- The implementation answers exact-binding questions semantically rather than by text search.
- Diagnostics preserve source provenance and are shaped for later unified diagnostic summaries.
- The slice composes with the existing AST `scan` surface rather than replacing it.

## Depends on
- the shared read-oriented query/result contract and adapter discovery seam
