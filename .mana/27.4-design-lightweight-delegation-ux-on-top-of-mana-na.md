Goal: make delegation in imp feel simple and obvious to a human operator while preserving mana as the durable work substrate and orchestration source of truth.

Current state:
- imp already has mana-native orchestration work and a stronger substrate than opencode.
- The product gap is UX: delegation should feel like an easy top-level action rather than requiring the user to think in raw mana graph operations.

Steps:
1. Inspect the current native mana tool, imp run entrypoints, and startup/discoverability surfaces.
2. Define the minimum delegation UX verbs and flows a human should get (for example: spawn helper, continue helper, fan out a subtask, inspect helper result, promote helper output into mana notes/unit updates).
3. Specify which actions belong in the native mana tool, which belong in `imp run`, and which belong in TUI discoverability/startup surfaces.
4. Capture follow-on implementation slices under the correct existing epics/jobs (27, 28.1, 29) instead of inventing a parallel delegation system.

Files/areas:
- imp/crates/imp-core/src/tools/mana.rs
- imp/crates/imp-cli/src/main.rs
- imp/crates/imp-tui/src/app.rs
- imp/crates/imp-tui/src/views/welcome.rs
- imp/crates/imp-tui/src/views/command_palette.rs

In scope:
- user-facing delegation verbs and flows
- CLI/TUI/native-mana-tool responsibility split
- resumability and result visibility expectations

Out of scope:
- replacing mana with session-only delegation
- deep runtime refactors unrelated to delegation UX

Do not:
- design a second orchestration substrate beside mana
- hide durable delegated work only in chat/session state
