  Repo-grounded findings:
  1. Short-term hardening already partially exists but is incomplete.
  - `.6.6` already closed the easiest Lua bypasses: `imp.tool()` can be disabled and `imp.env()` is deny-by-default unless allowlisted.
  - However, the actual runtime wiring is still incomplete: `imp-lua::init_lua_extensions()` does not thread an explicit capability policy from `imp-core`/builder/config into the Lua runtime.
  - `imp.exec`, `imp.http.*`, `imp.secret`, and `imp.secret_fields` remain broadly exposed in `bridge.rs`, so the current hardening is narrower than a real extension permission model.

  2. Medium-term guardrail work is cleanly represented by the existing 31.x units.
  - `guardrails.rs` and `config.rs` already contain the config/profile scaffolding and after-write hooks path.
  - 31.2/31.3/31.4 remain the correct medium-term follow-through for config, prompt/runtime integration, and built-in profile/docs polish.
  - This guardrail track is separate from Lua capability policy even though both are 'policy' concerns.

  3. Hook execution is part of the near-term trust boundary.
  - `hooks.rs` runs configured shell commands directly, including non-blocking background hooks.
  - This is already an explicit capability/trust surface and should stay in the near-term hardening bucket rather than being deferred entirely to guest-runtime redesign.
  - No obvious missing backlog item was created here because the current unit scope was reconciliation-only and the broader runtime policy epic 47.4 will likely subsume hook-boundary determinism.

  4. Longer-term architecture is correctly represented by 44.
  - `44` and `44.1` already capture the guest-runtime substrate direction: Rust host ownership, manifest/capability vocabulary, Lua as first guest runtime, future runtimes later.
  - Do not try to solve full extension substrate design in the short-term hardening jobs.

  5. Missing near-term tracked gap identified and created.
  - Created `46.3.1` to thread and enforce Lua capability policy through real runtime wiring, because current hardening primitives exist in `sandbox.rs`/`bridge.rs` but are not yet fully wired/configured at the builder/runtime boundary.

  Backlog split:
  - Near-term hardening: `.6.6` plus new `46.3.1` (real runtime wiring + broader exposed Lua host capabilities).
  - Medium-term guardrails: `31.2`, `31.3`, `31.4`.
  - Longer-term extension architecture: `44`, `44.1`, `44.2`, `44.3`.

  Conclusion:
  - 46.3 is now reconciled into an explicit three-layer backlog with one concrete missing short-term job added (`46.3.1`). No additional critical untracked policy gap was left implicit from this inspection.
