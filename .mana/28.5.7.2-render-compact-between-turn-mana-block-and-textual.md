## Task
Wire the canonical turn-batched mana review packet into imp-tui as the first shippable visual surface: a compact widget-tray block plus a lightweight expanded textual inspector.

## Goal
Replace raw per-action mana review noise with one between-turn review block in the TUI while reusing existing sidebar/detail surfaces.

## Grounded context
- `imp/crates/imp-tui/src/app.rs` currently renders a fixed-height widget tray (`widget_height = 5`) as a bordered paragraph titled `widgets`, concatenating keyed line widgets.
- `imp/crates/imp-tui/src/views/sidebar.rs` already provides stream/split sidebar modes and a line-oriented detail pane with cached render + plain-text surfaces.
- `docs/rebuild/imp-between-turn-mana-review-ui.md` constrains the first ship to a line-oriented compact block in the widget tray and a lightweight textual inspector, not a graph browser.
- Shared formatting/config and canonical TurnManaDelta input should come from sibling dependencies rather than being re-derived inside the TUI.

## Required behavior
1. Show a compact between-turn `mana` block in the widget tray from the canonical TurnManaDelta renderer.
2. Support review modes `off`, `compact_on_change`, and `pinned` with the documented auto-show behavior.
3. Preserve explicit `no_change`, `changed`, and `needs_decision` state rendering.
4. Reuse existing sidebar/detail surfaces for the expanded textual inspector of recent touched units, subtree context, open decisions, and dependency changes.
5. Auto-open or otherwise foreground the inspector only when the documented gating says to, especially for consequential unresolved choices.
6. Stop treating per-action mana widget updates as the primary between-turn review signal.

## Files
- `docs/rebuild/imp-between-turn-mana-review-ui.md` (read)
- `imp/crates/imp-tui/src/app.rs` (modify — widget tray + inspector control behavior)
- `imp/crates/imp-tui/src/views/sidebar.rs` (modify — textual inspector rendering path if needed)
- shared imp-core TurnManaDelta formatter/config from dependencies (read/use)

## Acceptance criteria
- TUI shows a compact between-turn mana review block in the widget tray for changed turns.
- TUI can show the expanded textual inspector using existing line/detail surfaces.
- `needs_decision` visibly foregrounds consequential unresolved choices without requiring a graph browser.
- Per-action mana tool chatter is no longer the primary between-turn review affordance.

## Do not
- Do not introduce a component-widget-only solution.
- Do not require fullscreen ownership or a graph canvas.
- Do not re-derive TurnManaDelta semantics from raw action-local tool details inside the TUI.
