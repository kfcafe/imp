## Task
Align the current `imp chat` `:view` handoff implementation with the explicit shell-to-view boundary documented in `docs/rebuild/imp-view-shell-handoff-patterns.md`.

## Current state
The first shell handoff slice is implemented in `imp/crates/imp-cli/src/main.rs`, but it still behaves like an inline reuse of the view renderer instead of a clearly bounded viewer launch:
- `ChatShellCommand::View(area)` currently prints `opening viewer: {area}`
- it then calls `run_view_mode(cli, Some(area.as_str())).await?` directly
- there is no explicit `Returned from viewer...` line after the view output completes

This is good enough for the first proving slice, but it does not yet match the documented boundary contract.

## Goal
Keep the existing `:view` parser and focused `imp view` areas, but make the shell handoff read as an explicit `imp view ...` launch with a visible return boundary.

## Steps
1. Open `imp/crates/imp-cli/src/main.rs` and locate `ChatShellCommand::View(area)` plus `run_view_mode()`.
2. Replace the generic lowercase `opening viewer: {area}` output with explicit target-aware launch text such as:
   - `Opening \`imp view sessions\`...`
   - `Opening \`imp view tree\` for the current session...`
   - `Opening \`imp view logs\` for the current session...`
   - `Opening \`imp view checkpoints\` for the current session...`
3. After the viewer path returns, print a compact return line. For the current plain-text viewer slice, `Returned from viewer: no selection` is acceptable for read-only areas until real selection payloads exist.
4. Keep the handoff code centralized. If helpful, add a small helper in `main.rs` or a tiny `imp-cli` helper module so the launch/return behavior is the single place to evolve later into a separate child-process invocation.
5. Preserve current focused area mappings and unknown-area help; do not broaden into a generic fullscreen authoring mode.
6. Update or add narrow tests only where needed so parser behavior and handoff strings remain covered.

## Files
- `imp/crates/imp-cli/src/main.rs` (modify — handoff messaging and return behavior)
- optional small helper under `imp/crates/imp-cli/src/` if needed, but avoid a wide refactor
- `docs/rebuild/imp-view-shell-handoff-patterns.md` (read only for contract)

## Acceptance criteria
- `:view ...` still routes to the focused viewer areas already implemented.
- The shell prints an explicit `imp view ...` launch cue before handing off.
- The shell prints an explicit return cue after the viewer path completes.
- The user-facing behavior does not silently morph the shell into a browser.
- This remains a narrow implementation slice; no interactive viewer redesign is required here.

## Non-goals
- full interactive viewer filtering/search
- converting the handoff to a fully separate process if that is larger than this slice
- changing `imp tui`
- re-opening the shell vs viewer product boundary decision
