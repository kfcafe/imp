Make the one-command source install flow (`uu install`, and by extension the repo-root source install path) repair the active shell command automatically instead of requiring a separate manual `imp install-local` step. Inspect how `uu install` targets the repo root today, then choose the narrowest way to hook the active-path repair into that flow. Options include making the root install shim run the install-local repair after launch in install mode, adding a small post-install helper script that `uu install` can invoke, or another equivalent repo-local mechanism. Keep scope to making the source install UX one-step and deterministic for users who type `imp` after install. Do not redesign packaging broadly.
