## Goal
Define and implement the shared imp-side runtime contract for Phase 1 read-oriented code intelligence.

## Current state
- `docs/design/imp-read-oriented-code-intelligence-plan.md` now fixes the model-facing shape: normalized locations, ranges, symbols, diagnostics, summaries, provenance, and freshness instead of raw protocol payloads.
- `imp/crates/imp-core/src/tools/scan/types.rs` already provides a typed AST-backed structural precedent (`ScanResult`, `TypeInfo`, `FunctionInfo`).
- No shared read-oriented semantic result contract exists yet for diagnostics, definition, references, symbols, hover, or signature help.

## Scope
- imp-owned query categories and result envelope types
- adapter/backend discovery seam
- fallback/provenance/freshness representation
- no write-oriented semantic actions

## Acceptance
- There is a clear shared imp-local contract another implementation slice can build on.
- The contract preserves the AST structural layer separately from the semantic binding/meaning layer.
- The contract leaves protocol choice open behind stable normalized shapes.

## Follow-on slices that depend on this
- diagnostics / go-to-definition / references
- symbols
- hover / signature help
- unified diagnostic summary and session UX
