## Goal
Add symbol-oriented read surfaces that complement the existing AST structural layer without collapsing the two.

## Current state
- The plan doc explicitly recommends `document symbols` and `workspace symbols` in Phase 1.
- The repo already has AST-backed `scan` that is strong for file shape and extraction.

## Scope
- document symbols
- workspace symbols
- explicit orientation/fallback rules between AST-backed `scan` and semantic symbol queries
- no semantic writes

## Acceptance
- The runtime can use symbols as a semantic table of contents while still preferring AST `scan` for fast local structure and extraction.
- The implementation documents and enforces when AST-first orientation should be used before heavier semantic lookups.
- Symbol results use the shared normalized location/range/summary/provenance contracts.
