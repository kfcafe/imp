## Goal
Turn `docs/design/imp-read-oriented-code-intelligence-plan.md` into a visible implementation rollout for Phase 1 read-oriented code-intelligence in `imp`.

## Current state
- The plan doc now fixes the Phase 1 read surface set: `diagnostics`, `go-to-definition`, `find references`, `document/workspace symbols`, `hover`, and `signature help`.
- The repo already has imp-owned AST-backed structural inspection via `imp/crates/imp-core/src/tools/scan/`.
- Runtime orchestration remains imp-owned; `mana` should not own semantic server lifecycle, document sync, or live query execution.

## Scope
Implementation decomposition only. No write-oriented semantic refactors in this epic.

## Required implementation slices
1. normalized query/result contracts and adapter discovery
2. diagnostics + go-to-definition + find references
3. document/workspace symbols plus AST-first orientation flow
4. hover + signature help
5. unified diagnostic summary plus compact CLI/TUI transcript UX

## Non-goals
- rename/code actions/import fixups
- collapsing AST and semantic layers into one fuzzy abstraction
- leaking raw protocol payloads to the model as the primary runtime contract
- shifting runtime orchestration into `mana`

## Done
Done when child slices exist in root mana with clear sequencing and each slice inherits the concrete decisions from the plan doc rather than reconstructing them from chat.
