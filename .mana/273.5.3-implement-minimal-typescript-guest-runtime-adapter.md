Goal: make imp execute a minimal subset of TypeScript extensions through a Rust-host-owned guest runtime adapter, with Pi compatibility layered on top rather than baked into the generic runtime.

Updated architecture:
- Generic TypeScript extension runtime: discovery under `.imp/extensions`, runner interface, host registry integration, callback execution, schema normalization, policy/error/cancellation/timeout behavior.
- Pi compatibility adapter: Pi import shape, `package.json.pi.extensions`, default-export `ExtensionAPI` shim, Pi result/schema/viewer mappings, and Pi common import shims.
- Real runtime should use Bun for `.ts` entrypoints. Current Node bridge is only a spike/proof and should be replaced or explicitly demoted.

Minimum compatibility target:
- Load extension entrypoints discovered under `.imp/extensions`.
- For Pi-compatible packages, support default-export function receiving a Pi-compatible `ExtensionAPI` shim.
- Capture `pi.registerTool({ name, description, parameters, execute })`.
- Register resulting tools into imp's host registry.
- Invoke registered `execute` callbacks when the agent calls the tool.
- Convert returned text/details into imp's tool result format.
- Normalize errors, cancellation/timeouts, unsupported API calls, and missing Bun.

Initial API stance:
- `registerTool`: supported.
- `registerCommand`: accept and record or warn/stub if command wiring is too much for first pass.
- `pi.on`: support a tiny lifecycle subset only if cheap; otherwise warn clearly for unsupported events.
- UI/native tool/env/secrets/process/network/filesystem capabilities remain host-mediated and fail closed until explicitly mapped.

Fixture target:
- Simple generic TS extension fixture.
- Pi compatibility fixture using TypeBox-style `registerTool`.
- Official Pi `dynamic-tools.ts` or a reduced equivalent for compatibility testing once Bun runner lands.

Out of scope:
- Full Pi API parity.
- Complex package-local dependency handling unless Bun makes it straightforward.
- Marketplace or remote install UX.
