Allow `imp chat --no-tools` to start without requiring provider auth when the user only needs shell-local commands. Current evidence from 2026-04-09: the new `:settings` shell command exists, but smoke verification inside `imp chat` is blocked because chat startup resolves provider/auth before the shell prompt appears, failing with missing Anthropic API key even in `--no-tools` mode. Goal: make the CLI-first shell usable for local command surfaces like `:settings`, `:help`, and future local setup/config commands without requiring remote model auth up front.

Scope:
1. inspect `imp-cli` chat startup and any relevant `ImpSession` creation/auth/model requirements
2. implement the smallest correct behavior change so `imp chat --no-tools` can enter the shell without provider auth
3. keep normal prompt execution honest: if the user actually sends a prompt that requires a model/provider, surface the error then rather than pretending the model path works offline
4. verify with narrow checks plus a smoke run that reaches the shell and exercises `:settings`

This is a CLI-first usability/verification unblocker, not a broad offline mode redesign.
