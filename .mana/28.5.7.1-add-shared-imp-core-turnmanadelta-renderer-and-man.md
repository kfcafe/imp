## Task
Add the shared imp-core formatting/config seam for rendering canonical between-turn mana review packets before wiring any frontend-specific surface.

## Goal
Keep TUI and CLI/RPC as adapters over one presentation-neutral TurnManaDelta rendering policy and one user control setting.

## Grounded context
- `docs/rebuild/imp-between-turn-mana-review-ui.md` specifies that the first ship renders canonical `TurnManaDelta` semantics, not raw per-action mana tool updates.
- `docs/rebuild/imp-between-turn-mana-review-ui.md` defines three control modes: `off`, `compact_on_change`, and `pinned`.
- `imp/crates/imp-core/src/config.rs` already owns shared UI config types, so the review mode belongs there rather than in a TUI-only setting.
- `imp/crates/imp-core/src/tools/mana.rs` currently emits per-action widget strings, but 28.5.6 is adding the canonical turn-batched packet in imp-core.

## Required behavior
1. Introduce a shared config enum/value for mana review display mode with explicit `off`, `compact_on_change`, and `pinned` semantics.
2. Add shared formatting helpers that take canonical `TurnManaDelta` input and produce:
   - compact line-oriented summary lines for the widget/footer surface
   - expanded line-oriented textual inspector lines
3. Preserve explicit `no_change`, `changed`, and `needs_decision` state rendering.
4. Preserve note/decision event semantics instead of collapsing them into generic update noise.
5. Keep the output presentation-neutral enough that TUI and CLI/RPC can both consume it.

## Files
- `docs/rebuild/imp-between-turn-mana-review-ui.md` (read for exact UI semantics)
- `docs/rebuild/mana-turn-delta-contract.md` (read for canonical input contract)
- `imp/crates/imp-core/src/config.rs` (modify — add shared mana review mode config)
- `imp/crates/imp-core/src/...` (modify/create — shared formatter module near the canonical TurnManaDelta types from 28.5.6)
- `imp/crates/imp-core/src/ui.rs` (read only unless a small shared adapter type is justified)

## Acceptance criteria
- Shared imp-core code can render compact between-turn mana review lines from canonical TurnManaDelta without depending on TUI widgets.
- Shared imp-core code can render expanded textual inspector lines from canonical TurnManaDelta without depending on TUI sidebar types.
- Config contains an explicit review mode enum/value for `off`, `compact_on_change`, and `pinned`.
- `needs_decision` formatting clearly surfaces consequential unresolved choices.

## Do not
- Do not implement frontend-specific layout logic here.
- Do not fall back to raw ToolOutput.details parsing in the adapters.
- Do not require a graph browser or component widget system.
