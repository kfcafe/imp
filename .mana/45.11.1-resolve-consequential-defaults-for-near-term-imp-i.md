Decision groups to resolve:
1. Worker/runtime reliability defaults
   - auth precedence for direct/headless `imp run`

2. Context efficiency defaults
   - auto-compaction default posture
   - preserved-tail guarantee for auto-compaction
   - truncation bias toward cheaper context vs more raw output retention
   - visibility policy for compaction/truncation events
3. Extension/script boundary defaults
   - script tool versus durable extension split
   - confirm Lua shipped now, TS/JS future-facing only
   - capability defaults for filesystem/network/subprocess/native-tool access
   - whether scripts can call native tools directly or only safe host APIs
   - ephemeral script versus durable extension boundary
