Implement a program-agnostic default-command layer in `uu` for local source installs. Keep existing ecosystem-native install behavior, but add an explicit `uu install --default` path that, after the normal install step, makes one or more executables become the default user-shell command in a preferred user-local bin directory (for example `~/bin` or `~/.local/bin`). Start with the narrowest robust slice for Cargo projects and repos like imp, but design the abstraction so other ecosystems can participate without forcing system-wide installs.

Execution outline:
1. Inspect uu install command parsing and project detection to add a post-install default-command phase.
2. Define policy: explicit flag, user-local target only, no sudo/system path writes by default, and clear conflict/override behavior.
3. For Cargo projects, infer or declare executable names and make them the shell-default command via copy or symlink semantics that are easy to reason about.
4. Add verification/reporting so uu tells the user whether `command -v <bin>` now resolves to the desired default path.
5. Prefer this general mechanism over repo-specific hooks once the primitive exists.

Non-goals:
- forcing system-wide install paths
- broad package-manager redesign
- opaque destructive PATH mutation
